Create the agreement creation page and builder component.

**`src/app/(app)/agreements/new/page.tsx`:**
Server component that fetches customers (with pools), checklist templates, agreement templates, and org_settings. Passes as props to AgreementBuilder. Same pattern as quote creation pages.

**`src/components/agreements/agreement-builder.tsx`:**
A scrollable single-page form (not a wizard — simpler for the typical 2-3 minute creation flow). Sections:

1. **Template Selection** (optional) — Dropdown to select a template. When selected, pre-fills term_type, frequency, pricing, and terms/conditions. "Start from scratch" is the default. Template selection populates defaults but everything remains editable.

2. **Customer Selection** — Combobox searching customers by name. On select, show customer's pools as a checklist. User checks which pools to include.

3. **Pool Configuration** — For each selected pool, render a `PoolEntryForm` component. Each pool entry has:
   - Frequency selector (weekly, biweekly, monthly, custom)
   - Custom interval days input (shown only when frequency = 'custom')
   - Preferred day of week selector
   - Pricing model radio group: Monthly Flat Rate, Per-Visit, Tiered
   - Monthly Flat: single amount input
   - Per-Visit: single amount input
   - Tiered: threshold visits input, base amount per visit, overage amount per visit
   - Service checklist: multi-select from org's checklist tasks (pulled from existing service requirements system). Each task is a checkbox with name.
   - Notes textarea

   Use the controlled decimal input pattern from MEMORY — local string state for inputs, flush on complete values only.

4. **Agreement Terms** — Term type selector (month_to_month, 6_month, 12_month). Start date (date picker). End date auto-calculated from term_type (or null for month_to_month). Auto-renew toggle.

5. **Terms & Conditions** — Rich text areas for: Terms & Conditions, Cancellation Policy, Liability Waiver. Pre-filled from template if one was selected, or from org default text. Editable.

6. **Internal Notes** — Textarea for office-only notes (not shown to customer).

7. **Summary** — Total monthly cost (sum of all pool entries), number of pools, frequency breakdown.

8. **Actions** — "Save as Draft" button (creates with status='draft'), "Save & Send" button (creates then triggers send flow — Plan 03 will wire the actual send).

Form validation: customer required, at least one pool selected, each pool must have pricing filled, start_date required. Use plain React state validation (NOT zodResolver — incompatible with zod@4 per MEMORY).

On submit: call `createAgreement()` server action. On success, redirect to `/agreements/${id}` (detail page built in Plan 05). Show toast on success/error.

**`src/components/agreements/pool-entry-form.tsx`:**
Reusable form component for a single pool entry within the builder. Receives pool info (name, type, volume) and checklist tasks. Renders frequency, pricing model, checklist, and notes. Emits changes up to parent via callback.

Match existing design system: Card components, consistent gap-6 between sections, gap-4 within, p-5 card padding, sm button sizes, outline variants. Dark-first design. No icons next to section headers.

**`src/components/settings/agreement-templates-tab.tsx`:**
Settings tab for managing agreement templates. Contains:

1. **Template List** — Shows all templates as cards (or table rows). Each shows: name, default term type, default frequency, default pricing model, active/inactive badge. Clickable to edit.

2. **Create/Edit Template Dialog** — Modal form with:
   - Name (text input, required)
   - Default term type (select: month_to_month, 6_month, 12_month)
   - Default frequency (select: weekly, biweekly, monthly)
   - Default pricing model (select: monthly_flat, per_visit, tiered)
   - Default monthly amount (numeric input)
   - Terms & Conditions (textarea — the full default legal text)
   - Cancellation Policy (textarea)
   - Liability Waiver (textarea)
   - Service Description (textarea)
   - Active toggle

   Provide sensible default text for Terms & Conditions, Cancellation Policy, and Liability Waiver out of the box — pool service industry standard language. This is the DeweyIQ-provided default that companies customize.

   Default T&C should cover: service obligations, customer responsibilities (access, chemical top-up), payment terms, dispute resolution. Keep it professional but concise (not overly legal).

   Default Cancellation Policy: "{notice_period_days} days written notice required. Early termination may apply for term contracts."

   Default Liability Waiver: Standard limitation of liability for pool service — property damage from pre-existing conditions, chemical balance disclaimers, force majeure.

3. **Delete** — Delete button on each template. Disabled if template is referenced by active agreements. Show count of agreements using it.

4. **Agreement Defaults Section** — Below the template list, show org_settings fields:
   - Cancellation notice period (days input, default 30)
   - Renewal reminder lead days (multi-value, e.g. 30 and 7 days before)
   - Agreement number prefix (text input, default "SA")
   These update org_settings directly.

Wire to server actions: getAgreementTemplates, createAgreementTemplate, updateAgreementTemplate, deleteAgreementTemplate. Server actions return fresh data — set state directly (don't rely on router.refresh per MEMORY).
