- term_type (text, NOT NULL) — values: 'month_to_month', '6_month', '12_month'
- start_date (text) — YYYY-MM-DD format
- end_date (text, nullable) — null for month_to_month
- auto_renew (boolean, default true)
- template_id (uuid, nullable) — FK → agreement_templates
- terms_and_conditions (text)
- cancellation_policy (text)
- liability_waiver (text)
- internal_notes (text) — office-only
- version (integer, default 1)
- sent_at (timestamp with time zone)
- signed_at (timestamp with time zone)
- declined_at (timestamp with time zone)
- cancelled_at (timestamp with time zone)
- paused_at (timestamp with time zone)
- paused_reason (text)
- renewed_at (timestamp with time zone)
- signature_name (text) — typed name
- signature_image_base64 (text) — canvas draw PNG

- decline_reason (text)

- checklist_task_ids (jsonb, default '[]') — array of task IDs

- schedule_rule_id (uuid, nullable) — FK → schedule_rules, set on acceptance

Create `src/actions/agreements.ts` with server actions following the exact pattern from `src/actions/quotes.ts`:

**Agreement CRUD:**
- `createAgreement(data)` — Creates a new service agreement with pool entries. Accepts customer_id, term_type, start_date, end_date, auto_renew, template_id (optional), terms_and_conditions, cancellation_policy, liability_waiver, internal_notes, and pool_entries array. Auto-generates agreement_number from org_settings (SA-0001 format, incrementing next_agreement_number). Creates the service_agreements row AND all agreement_pool_entries rows in a single withRls transaction. Returns the created agreement with pool entries.

- `getAgreements(filters?)` — Returns all agreements for the org. Accepts optional filters: status (string or string[]), customer_id, search (matches agreement_number or customer name). Uses Drizzle relational query with `with: { customer: true, poolEntries: { with: { pool: true } } }`. Returns sorted by created_at desc.

- `getAgreement(id)` — Returns a single agreement with all relations: customer, poolEntries (with pool), amendments, template. Uses withRls.

- `updateAgreement(id, data)` — Updates agreement fields. Does NOT allow updating if status is 'active' (use amendment flow instead). Appends to activity_log.

- `deleteAgreement(id)` — Only allows deleting drafts. Soft check: status must be 'draft'.

- `sendAgreement(id)` — Updates status to 'sent', sets sent_at. This action is called by Plan 03 (email delivery) — for now, just handle the status transition and activity log.

**Template CRUD:**
- `getAgreementTemplates()` — All templates for org, sorted by name
- `createAgreementTemplate(data)` — Create new template
- `updateAgreementTemplate(id, data)` — Update template
- `deleteAgreementTemplate(id)` — Delete template (only if not referenced by active agreements — check with a count query first)

All actions use `"use server"` directive, `withRls(token, ...)` pattern with `getAuthToken()`, proper error handling returning `{ success, data?, error? }` pattern. Follow the exact same patterns as `src/actions/quotes.ts` for auth, error handling, and return types.

IMPORTANT: When building the agreement number, use `toLocalDateString()` from `@/lib/date-utils` if any date logic is needed — NEVER `new Date().toISOString().split("T")[0]`.
