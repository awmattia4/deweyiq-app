**`src/app/api/agreements/[id]/sign/route.ts`:**
POST handler for accepting or declining agreements. Uses `adminDb` (customer has no session).

**Request body:** `{ action: 'accept' | 'decline', signatureName?, signatureImageBase64?, declineReason?, token }`

**Token verification:** Verify the token via `verifyAgreementToken(body.token)`. If invalid, return 401. Verify the decoded agreementId matches the route param `id`.

**Idempotency guard:** If `agreement.status !== 'sent'`, return 409 with `{ error: "This agreement has already been processed" }`. Same pattern as quote approve handler.

**Accept flow:**
1. Capture metadata: IP from `req.headers.get('x-forwarded-for')` or `req.ip`, user agent from `req.headers.get('user-agent')`, timestamp.

2. Update service_agreements via `adminDb`:
   - status = 'active'
   - signed_at = now
   - signature_name = body.signatureName
   - signature_image_base64 = body.signatureImageBase64 (if provided)
   - signature_ip = ip
   - signature_user_agent = userAgent

3. **Auto-provision schedule rules (AGREE-05):** For each `agreement_pool_entry`:
   - Insert into `schedule_rules`: org_id, customer_id (from agreement), pool_id, frequency, anchor_date = pool_entry.start_date or agreement.start_date (use `toLocalDateString` if computing — but prefer using the stored date string directly), preferred_day_of_week, checklist_template_id (if applicable), active = true.
   - Update the pool_entry's `schedule_rule_id` with the created rule's ID.

4. **Auto-configure billing:** Update the customer's billing model based on the pool entries:
   - If all entries are `monthly_flat`: set `customers.billing_model = 'flat_rate'`, `flat_rate_amount` = sum of all monthly amounts.
   - If all entries are `per_visit`: set `customers.billing_model = 'per_visit'`.
   - If mixed or tiered: set to `flat_rate` with the total monthly equivalent. (Tiered pricing is tracked in the agreement but billing uses the equivalent monthly rate for simplicity in Phase 14.)

5. Append to activity_log: `{ type: "agreement_signed", at: timestamp, by_id: null, note: "Customer signed agreement. Signed as: {signatureName}" }`.

6. Return 200 with `{ success: true }`.

**Decline flow:**
1. Update: status = 'declined', declined_at = now, decline_reason = body.declineReason
2. Append to activity_log: `{ type: "agreement_declined", at: timestamp, note: "Customer declined. Reason: {reason or 'No reason provided'}" }`
3. Return 200 with `{ success: true }`

**Update `src/app/api/agreements/[id]/pdf/route.ts`:**
Add token-based authentication as a fallback: if no authenticated session, check for `?token=` query param. Verify via `verifyAgreementToken`. If valid AND matches the agreement ID, serve the PDF. This allows the public approval page to link to PDF download.

**Error handling:** All DB operations wrapped in try/catch. Return 500 with generic error on failure. Log actual error server-side.
