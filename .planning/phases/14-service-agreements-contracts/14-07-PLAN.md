**Add `runAgreementRenewalScan()` to `src/actions/agreements.ts`:**

This function scans ALL orgs (uses `adminDb` since it's cron — no user session):

1. Get all active agreements where:
   - status = 'active'
   - end_date IS NOT NULL (month-to-month doesn't expire)
   - auto_renew = true OR auto_renew = false (both get reminders — but the message differs)

2. For each org, read `agreement_renewal_lead_days` from org_settings (default [30, 7]).

3. For each agreement, calculate days until expiration: `end_date - today`.

4. If days_until_expiry matches any value in the lead_days array AND the `renewal_reminder_sent_at` is null or was set more than 24 hours ago (prevents duplicate sends within the same reminder window):
   - Send renewal reminder email to the office (org owner/office users — NOT the customer)
   - Update `renewal_reminder_sent_at` = now
   - Append to activity_log: "Renewal reminder sent — {days} days until expiry"

5. Track counts and return summary.

**Add compliance check logic to `src/actions/agreements.ts`:**

**`getAgreementCompliance(agreementId)` server action:**
For each pool_entry in an active agreement, check compliance over a rolling 30-day window:

1. **Service frequency compliance:**
   - Calculate expected stops in the last 30 days based on frequency:
     - Weekly: ~4 stops
     - Biweekly: ~2 stops
     - Monthly: ~1 stop
     - Custom: 30 / custom_interval_days
   - Count actual completed route_stops for this pool_id in the last 30 days (where `completed_at IS NOT NULL`). Exclude stops where `skipped = true` or there's a valid skip_reason.
   - If actual < expected - 1 (allow 1 miss buffer): flag as `frequency_breach`
   - If 2+ consecutive misses: flag as `critical_breach`

2. **Billing compliance:**
   - For monthly_flat pricing: sum invoices for the customer in the current billing period. Compare against agreement's monthly_amount. If mismatch > $1 tolerance: flag as `billing_mismatch`.
   - For per_visit: count completed stops * per_visit_amount, compare against invoiced total.
   - For tiered: calculate based on threshold and overage rates.

3. Return compliance summary per pool entry:
   ```
   {
     pool_id, pool_name,
     expected_stops, actual_stops,
     frequency_status: 'compliant' | 'warning' | 'breach',
     billing_status: 'compliant' | 'mismatch',
     details: string (explanation)
   }
   ```

Only check agreements with status = 'active' (not paused, cancelled, etc.). This avoids false positives during pauses per Research pitfall #6.

**`getAgreementsWithCompliance()`:**
Bulk version that fetches all active agreements with compliance data. Used by the agreement manager for the list view. For performance, use a single query with LEFT JOINs and GROUP BY rather than per-agreement queries.
