**`src/lib/pdf/agreement-pdf.tsx`:**
Create the agreement PDF document using @react-pdf/renderer. Follow the exact same pattern as `src/lib/pdf/quote-pdf.tsx` — createElement-based rendering, hex colors only (NO oklch), server-compatible.

PDF sections (pool service industry standard structure):

1. **Header** — Company logo (if set) + company name, "Service Agreement", agreement number (e.g. SA-0001), date created, term dates (start – end or "Month-to-Month")

2. **Parties** — Company: name, address, phone, email. Customer: name, service address, phone, email.

3. **Scope of Service** — For each pool entry: pool name, pool type (pool/spa/fountain), frequency (e.g. "Weekly — Every Monday"), service checklist table (task names in a bulleted list).

4. **Pricing & Billing** — For each pool entry: pricing model description.
   - Monthly Flat: "$X.XX/month"
   - Per-Visit: "$X.XX per visit"
   - Tiered: "First N visits: $X.XX/visit, Additional visits: $Y.YY/visit"
   - Total monthly cost summary at bottom.

5. **Term & Renewal** — Start date, end date (or "Ongoing"), auto-renew status, renewal terms.

6. **Cancellation Policy** — From agreement's cancellation_policy field.

7. **Terms & Conditions** — From agreement's terms_and_conditions field. Render as paragraphs.

8. **Liability & Limitations** — From agreement's liability_waiver field.

9. **Signature Block** — Two-column layout: Company representative (blank line for name, date) and Customer (blank line for signature, printed name, date). Below: "This agreement is electronically signed via DeweyIQ. Signature, IP address, and timestamp are captured and stored securely."

10. **Footer** — "Powered by DeweyIQ" at bottom of each page. Page numbers.

Props interface: agreement object with customer, pool entries (with pool), org settings (company name, logo, address, etc.).

CRITICAL: All colors must be hex values. No oklch, no CSS variables. Use the same style patterns (font sizes, spacing, margins) as the quote PDF for visual consistency.

**`src/lib/agreements/agreement-token.ts`:**
Copy the exact pattern from `src/lib/quotes/quote-token.ts`. Two functions:

- `signAgreementToken(agreementId: string): Promise<string>` — Signs JWT with `{ agreementId }` payload, HS256, 180-day expiry (longer than quote's 90d to cover 12-month agreement terms). Uses `AGREEMENT_TOKEN_SECRET` env var.

- `verifyAgreementToken(token: string): Promise<{ agreementId: string } | null>` — Verifies and extracts agreementId. Returns null on any error (expired, invalid, tampered).

**`src/lib/emails/agreement-email.tsx`:**
Create React Email template following the exact pattern of existing email templates (e.g. `quote-email.tsx`). The email should:
- Company logo at top (if set)
- "Service Agreement Ready for Review" heading
- Customer name greeting
- Brief summary: "Your service agreement for [pool count] pool(s) is ready for review."
- Key terms summary: term type, start date, total monthly cost
- "Review & Sign Agreement" button linking to `/agreement/{token}` (the public approval page)
- "Download Full Agreement (PDF)" secondary link (links to a token-authenticated PDF route — same token works)
- Footer: company contact info + "Powered by DeweyIQ"

Props: customerName, companyName, companyLogo, agreementNumber, termType, startDate, totalMonthlyCost, poolCount, approvalUrl, pdfUrl.

**Update `src/actions/agreements.ts` — wire sendAgreement:**
Update the `sendAgreement(id)` action to:
1. Fetch the agreement with all relations (customer, pool entries)
2. Generate JWT token via `signAgreementToken(id)`
3. Generate the approval URL: `${process.env.NEXT_PUBLIC_APP_URL}/agreement/${token}`
4. Generate the PDF buffer (same as the PDF route — call renderToBuffer)
5. Send email via Resend with:
   - To: customer.email
   - Subject: "Service Agreement from {companyName} — SA-{number}"
   - React component: AgreementEmail with all props
   - Attachment: the PDF buffer as "Agreement-SA-{number}.pdf"
6. Update agreement: status = 'sent', sent_at = now
7. Append to activity_log: `{ type: "agreement_sent", at: timestamp, by_id: userId, note: "Agreement sent to {email}" }`
8. Return success with the updated agreement

Use `adminDb` for the Resend email send (it's a server action, but the email content is generated server-side). The status update uses `withRls` as usual.

IMPORTANT: The approval page URL uses `/agreement/[token]` (NOT `/api/agreements/...`). This is a public Next.js page, not an API route. Same pattern as `/quote/[token]`.
