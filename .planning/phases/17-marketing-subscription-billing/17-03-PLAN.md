  <action>
    Create `src/app/api/webhooks/stripe/route.ts`:

    1. Export async POST handler that:
       a. Reads raw body via `req.text()` (NOT req.json() — Stripe needs raw body for signature)
       b. Gets `stripe-signature` header
       c. Verifies signature with `stripe.webhooks.constructEvent(body, sig, STRIPE_WEBHOOK_SECRET)`
       d. Returns 400 on invalid signature
       e. Routes to handler based on event.type

    2. Event handlers (all use adminDb for writes):

       `handleCheckoutCompleted(session)`:
       - Extract customer ID and subscription ID from session
       - Look up subscription row by stripe_customer_id
       - If found: update stripe_subscription_id, status to "active"
       - Retrieve the Stripe subscription to get price, tier info, period dates
       - Update tier, stripe_price_id, current_period_start/end, pool_limit based on tier

       `handleSubscriptionUpdated(subscription)`:
       - Look up by stripe_subscription_id or stripe_customer_id
       - Map Stripe status to local enum (stripe "active" -> "active", "trialing" -> "trialing", etc.)
       - Determine tier from the price ID (reverse lookup against TIER_PRICE_IDS)
       - Update: status, tier, stripe_price_id, current_period_start, current_period_end, cancel_at_period_end, pool_limit, trial_ends_at (if trialing)
       - Set updated_at to now

       `handleSubscriptionDeleted(subscription)`:
       - Look up by stripe_subscription_id
       - Set status to "canceled", updated_at to now

       `handlePaymentSucceeded(invoice)`:
       - Look up by stripe_customer_id (from invoice.customer)
       - Reset: failed_payment_count to 0, last_payment_error to null, grace_period_ends_at to null
       - Set updated_at to now

       `handlePaymentFailed(invoice)`:
       - Look up by stripe_customer_id
       - Increment failed_payment_count
       - Set last_payment_error to invoice failure message
       - If failed_payment_count was 0 (first failure): set grace_period_ends_at to now + 7 days
       - Set updated_at to now

    3. Return `NextResponse.json({ received: true })` for all handled events
    4. Return `NextResponse.json({ received: true })` for unhandled event types too (don't 400 on unknown events)

    CRITICAL: This route must NOT require authentication — Stripe calls it directly.
    CRITICAL: Use adminDb for all writes — webhook handler operates at system level, not user level.
    CRITICAL: The handler must be idempotent. For subscription updates, always SET the full state (don't increment status). For payment failures, the increment is safe because Stripe sends distinct events per retry.

    Helper function `tierFromPriceId(priceId: string): BillingTier`:
    - Reverse lookup: iterate TIER_PRICE_IDS to find which tier/interval matches the price ID
    - Default to "starter" if not found (safety net)
  </action>
