    **1. `src/lib/unit-conversion.ts`** — Unit conversion utility:
    ```
    export function convertUnits(amount: number, fromUnit: string, toUnit: string): number
    ```
    Conversion table: 128 floz = 1 gallon, 16 floz = 1 cup, 32 floz = 1 quart, 16 oz = 1 lb. If units match, return amount unchanged. If conversion not found, return amount as-is (log warning). This is critical for matching dosing engine units (floz/lbs) to truck inventory units (gallons/lbs/each).

    **2. `src/actions/truck-inventory.ts`** — Full truck inventory server actions using `withRls` for user-facing queries:

    - `getTruckInventory(techId: string)` — Fetch all inventory items for a tech, ordered by category then item_name. Include joins to parts_catalog and chemical_products for display enrichment.
    - `addTruckInventoryItem(data)` — Insert new item. If barcode provided, also upsert into barcode_catalog_links.
    - `updateTruckInventoryItem(itemId, data)` — Update quantity, min_threshold, on_truck, etc. Log change in truck_inventory_log.
    - `deleteTruckInventoryItem(itemId)` — Soft or hard delete (owner/office only).
    - `decrementTruckInventoryFromDosing(techId, orgId, dosingAmounts, token)` — The core auto-decrement function:
      1. Load tech's truck inventory items matching `chemical_product_id IN (productIds)`.
      2. For each dosing amount, find matching inventory item.
      3. Use `convertUnits()` to handle unit mismatch (dosing in floz, inventory in gallons).
      4. Decrement quantity (clamp to 0).
      5. Log each change in truck_inventory_log with change_type='auto_decrement', source_type='service_visit'.
      6. If new quantity <= min_threshold AND reorder_alert_sent_at IS NULL: fire reorder alert via existing `alerts` table (alert_type='low_inventory', reference_type='truck_inventory') AND push notification via `notifyUser` to tech AND `notifyOrgRole('office')`. Set reorder_alert_sent_at.
      7. Return array of deducted items with before/after quantities for the confirmation prompt.
      CRITICAL: This function must NEVER throw — wrap entire body in try/catch, log errors, return empty array on failure.
    - `applyTruckLoadTemplate(techId, templateId)` — Copy all template items into truck_inventory for the tech. Skip items that already exist (match on catalog_item_id or chemical_product_id or item_name).
    - `transferInventoryItem(fromTechId, toTechId, itemId, quantity)` — One-sided transfer: decrement from source tech, create/increment on target tech. Log transfer_out on source, transfer_in on target. Use adminDb for cross-tech writes.
    - `getTruckLoadTemplates()` — Fetch all templates for org.
    - `createTruckLoadTemplate(data)` / `updateTruckLoadTemplate(id, data)` / `deleteTruckLoadTemplate(id)` — Template CRUD (owner/office).
    - `resetReorderAlert(itemId)` — Clear reorder_alert_sent_at when item restocked above threshold.

    **3. `src/actions/barcode.ts`** — Barcode server actions:
    - `resolveBarcode(barcode: string)` — Look up barcode in org's barcode_catalog_links. If found, return the linked item info. If not found, call `lookupBarcode` for UPC data.
    - `lookupBarcode(barcode: string)` — Call UPCitemdb API (`https://api.upcitemdb.com/prod/trial/lookup?upc=${barcode}`) with 24h cache. Return { name, category, found }.
    - `registerBarcode(barcode, itemData)` — Save barcode -> item mapping in barcode_catalog_links. Org-wide: once any tech scans, all techs recognize it.
