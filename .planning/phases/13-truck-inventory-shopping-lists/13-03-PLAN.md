  <action>
    **1. `src/actions/purchasing.ts`** — Purchasing dashboard and PO management:

    - `getPurchasingDashboard(groupBy: 'supplier' | 'urgency')` — Aggregate all shopping_list_items with status IN ('needed', 'ordered') across the org. Use LEFT JOIN + GROUP BY (never correlated subqueries in withRls).
      - Supplier grouping: group by vendor field (or "Unassigned" if null), show item count, total estimated cost.
      - Urgency grouping: group by is_urgent + status, show urgent-needed first, then needed, then ordered.
      Return: { groups: Array<{ key, label, items, itemCount, estimatedTotal }> }.

    - `createPurchaseOrder(data: { supplierName, supplierContact?, supplierEmail?, mode: 'formal'|'checklist', items: Array<{ shoppingListItemId?, itemName, quantity, unit, unitPrice }>, notes? })` — Insert into purchase_orders with auto-incrementing po_number from org_settings.next_po_number (same pattern as next_invoice_number). Create po_line_items. Calculate total_amount. Update linked shopping_list_items to status='ordered' with ordered_at, ordered_by_id, vendor, po_reference.

    - `sendPurchaseOrder(poId)` — For formal mode: generate PO PDF using @react-pdf/renderer (follow existing invoice/quote PDF pattern), send to supplier_email via existing email infrastructure. Set sent_at, update status to 'sent'.

    - `updatePurchaseOrderStatus(poId, status)` — Mark PO as partial/complete/cancelled. When complete: optionally transition all linked shopping_list_items to 'received'.

    - `getSpendingInsights(period: 'week'|'month'|'quarter', compareBy: 'tech'|'supplier'|'category')` — Aggregate purchase_orders + po_line_items for spending trends. Return time-series data for recharts: { timeSeries: Array<{ date, total }>, breakdown: Array<{ key, label, total, percentage }> }. Use LEFT JOIN + GROUP BY.

    **2. `src/actions/qbo-items.ts`** — QBO two-way item sync:

    - `syncCatalogItemToQbo(itemId)` — Read parts_catalog item, map to QBO Item fields (Name->name, Sku->sku, UnitPrice->default_sell_price, PurchaseCost->default_cost_price, Type->is_labor maps to Service/NonInventory). If item has qbo_item_id, update; else create and save qbo_item_id back. Use existing QBO client pattern from qbo-sync.ts.

    - `syncQboItemToDeweyIq(qboItemId, realmId)` — Called from webhook handler. Fetch QBO Item, find or create matching parts_catalog entry by qbo_item_id. Map fields in reverse.

    - `reconcileCatalogWithQbo(orgId)` — Bulk sync: fetch all QBO Items via findItems, match against parts_catalog by qbo_item_id or SKU. Create missing items in either direction. Return { created, updated, conflicts }.

    - `importQboItems(orgId)` — One-time import: pull all QBO Items into parts_catalog. Used from Settings. Returns count of imported items.

    **3. Update `src/lib/qbo/mappers.ts`:**
    Add `mapCatalogItemToQboItem(item)` and `mapQboItemToCatalogItem(qboItem)` mapper functions. Follow existing mapper pattern (mapCustomerToQboCustomer, etc.).

    **4. Update `src/app/api/qbo/webhook/route.ts`:**
    Extend the existing QBO webhook handler to listen for Item events (Item.Create, Item.Update, Item.Delete). When Item event received, call `syncQboItemToDeweyIq`. Follow existing webhook handler pattern for other entity types.
