**src/actions/projects-inspections.ts:**

**Inspections (PROJ-69):**
- `createInspection(token, projectId, data)` — Insert: inspection_type (building, electrical, plumbing, pool/spa, final, etc.), scheduled_date, inspector_name, inspector_contact, phase_id (optional), status='scheduled'.
- `recordInspectionResult(token, inspectionId, data)` — Update: status (passed/failed), actual_date, result_notes, documents (upload inspection reports).
- `getInspections(token, projectId)` — Fetch all inspections with status.

**Failed inspection rework (PROJ-70):**
- On inspection_status='failed': create correction tasks from inspection.correction_tasks JSONB. Insert as project_phase_tasks on the relevant phase with is_required=true. Assign to responsible party (tech or sub). Create alert for office.

**Quality checkpoints (PROJ-71):**
- `getQualityChecklist(token, phaseId)` — Return self-inspection checklist for the phase type. Configurable per phase template. Default items per phase type (Claude's discretion). These are checked by the tech BEFORE marking a phase complete. Export this function for use by completePhase.

**CRITICAL — Wire quality checklist validation into completePhase (PROJ-71):**
Update `completePhase` in `src/actions/projects-field.ts` — before allowing phase completion, call `getQualityChecklist(token, phaseId)` and validate that ALL returned checklist items are marked as completed. If any checklist items are unresolved/incomplete, return an error listing the incomplete items (e.g. `{ error: "Cannot complete phase: quality checklist items incomplete", incompleteItems: [...] }`). Do NOT allow the phase to be marked complete. Import `getQualityChecklist` from `./projects-inspections`.

**Punch list (PROJ-72):**
- `createPunchListItem(token, projectId, data)` — Insert: item_description, assigned_to, photo_urls.
- `updatePunchListItem(token, itemId, data)` — Update status (open -> in_progress -> resolved -> accepted), resolution_notes, photo evidence.
- `getPunchList(token, projectId)` — Fetch all punch list items with status.
- `customerSignOffPunchList(projectId, signatureData)` — Uses adminDb (called from portal). Customer accepts all resolved items. Sets customer_accepted_at on each accepted item. Triggers: stage='complete', warranty activation, final invoice generation.

**src/actions/projects-warranty.ts:**

**Warranty terms (PROJ-73):**
- `getWarrantyTerms(token, projectType)` — Fetch warranty terms for project type.
- `createWarrantyTerm(token, data)` — Insert: project_type, warranty_type (workmanship/equipment/surface), duration_months, what_covered, exclusions.
- `updateWarrantyTerm(token, id, data)` — Update term.

**Warranty activation:**
- `activateWarranty(token, projectId)` — Called on project completion (after walkthrough sign-off). Set project stage='warranty_active'. Calculate warranty expiration dates from warranty terms. Generate warranty certificate. Send to customer via email.

**Warranty certificate (PROJ-74):**
- `generateWarrantyCertificate(token, projectId)` — Generate PDF and email.

**src/lib/pdf/warranty-certificate-pdf.tsx:**
Warranty certificate PDF using @react-pdf/renderer. Claude's discretion on design:
- Company logo + name
- "WARRANTY CERTIFICATE" heading
- Certificate number (project number based)
- Customer name and property address
- Project description and completion date
- Warranty coverage table: type, duration, what's covered, exclusions, expiration date
- Terms and conditions
- Company authorized signature block
- "Powered by DeweyIQ" footer
HEX colors only. Professional, clean design.

**Warranty claims (PROJ-75, 76):**
- `submitWarrantyClaim(data)` — Uses adminDb (called from portal). Insert warranty_claim: project_id, warranty_term_id, customer_description, status='submitted'. Create alert for office.
- `reviewWarrantyClaim(token, claimId, data)` — Office reviews: approve/deny with notes. If approved and is_warranty_covered=true: create work_order with is_warranty_covered=true (PROJ-76 -- no invoice generated for covered work, cost absorbed by company). If not covered, create standard billable work order.
- `resolveWarrantyClaim(token, claimId, resolution)` — Set status='resolved', resolution_notes.

**Warranty expiration reminders (PROJ-77):**
- `checkWarrantyExpirations()` — Scheduled job (adminDb). Find projects in 'warranty_active' stage where any warranty term expires within 90/60/30 days. Send reminder email to customer. Create alert for office. Include offer to extend/renew.
