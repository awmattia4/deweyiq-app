- `createProjectPhase(token, projectId, data)` — create phase with name, sort_order, dependency_phase_id, dependency_type, estimated dates, assigned_tech_id, estimated_labor_hours, is_outdoor. Add to activity_log.
- `updateProjectPhase(token, phaseId, data)` — update phase fields. If dates change on a phase with dependents, cascade date recalculation (PROJ-43): fetch all phases for project, build dependency DAG, topologically sort, recalculate start/end dates for all downstream phases. Update project estimated_completion_date if final phase changes. Add to activity_log.
- `deleteProjectPhase(token, phaseId)` — soft-remove (update status to 'skipped' + archived_at). Recalculate dependents.
- `createPhaseTasks(token, phaseId, tasks[])` — bulk insert tasks with sort_order.
- `updatePhaseTask(token, taskId, data)` — update task completion status, notes.
- `checkStalledProjects(token)` — query projects where last_activity_at < now() - org_settings.project_inactivity_alert_days AND status='active'. Create alerts for stalled projects (using existing alerts table pattern). Default threshold: 7 days per research recommendation.
- `getProjectsForCustomer(token, customerId)` — fetch all projects for a customer (PROJ-79 archive access).
- `suggestServiceAgreement(token, projectId)` — check if customer has active service agreement; if not, return a flag to show "Offer Recurring Service" prompt (PROJ-78).
