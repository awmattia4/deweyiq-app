**src/actions/projects-scheduling.ts:**

- `updatePhaseDates(token, phaseId, newStartDate, newEndDate)` — Update phase dates. Then call cascadeDependencies for the project. Update project.estimated_completion_date based on latest phase end date. Add to activity_log. Update project.last_activity_at.

- `cascadeDependencies(token, projectId)` — CRITICAL: This is the core scheduling engine per research Pitfall 6.
  1. Fetch all phases for the project, sorted by sort_order.
  2. Build a dependency DAG from dependency_phase_id references.
  3. Topological sort to get execution order.
  4. For each phase with a dependency: if dependency is 'hard', phase.start = max(phase.start, predecessor.end + 1 day). If 'soft', log warning but don't force move.
  5. Apply the new dates in a single atomic transaction.
  6. If any downstream phases shifted, add activity_log entries noting the cascade.
  7. Update project.estimated_completion_date from the latest phase end date.
  8. Return the list of phases that were shifted (for Gantt UI refresh).

- `assignCrewToPhase(token, phaseId, techId)` — Assign tech to a project phase. Check tech availability: query route_stops for the same date range. If tech has service route conflicts, return warning (PROJ-42). Office can override. Update phase.assigned_tech_id.

- `checkWeatherDelay(token, projectId)` — For phases flagged as is_outdoor=true that are currently in_progress or starting within 7 days: check weather API (existing weather infrastructure from Phase 10) for severe conditions (rain, extreme heat, high wind). If bad weather predicted, create alert with suggested delay. Does NOT auto-delay -- office decides (PROJ-45).

- `holdProject(token, projectId, reason)` — Set status='on_hold', pause all in-progress phases. (May already exist from Plan 03 -- extend if needed.)
- `resumeProject(token, projectId)` — Set status='active', calculate new dates based on hold duration. Shift all remaining phases forward by the hold duration.

- `getGanttData(token, projectId)` — Fetch phases and format for SVAR Gantt: { tasks: GanttTask[], links: GanttLink[] }. Map project_phases to the SVAR format (id, text, start_date as Date, end_date as Date, progress based on task completion %). Map dependencies to links (source, target, type "0" for finish-to-start).
