**src/actions/projects-field.ts:**
- `getTechProjects(token)` — Fetch projects where current user is assigned_tech_id on at least one phase that is 'not_started' or 'in_progress'. Return project summary: name, address, current phase, today's tasks. Use LEFT JOIN pattern.
- `getProjectPhaseDetail(token, phaseId)` — Fetch phase with tasks, time logs, photos for the tech's view.
- `completeTask(token, taskId)` — Mark task as completed with completed_at and completed_by. Update project.last_activity_at.
- `uncompleteTask(token, taskId)` — Uncheck a task (in case of mistake).

**Time logging (PROJ-48):**
Per user decision: "Time logging supports both start/stop timer and manual entry. This is task-level time logging, distinct from shift-level clock-in/out. Project task time should reference the parent time_entry_id for reconciliation."
- `startProjectTimer(token, phaseId, taskId?)` — Create project_time_log with start_time=now(), entry_type='timer'. Find active time_entry (shift) for the tech and set time_entry_id for reconciliation.
- `stopProjectTimer(token, timeLogId)` — Set end_time=now(), calculate duration_minutes.
- `logManualTime(token, phaseId, data)` — Create project_time_log with entry_type='manual', duration_minutes, notes. Also set time_entry_id if shift is active.

**Photo capture (PROJ-50):**
Per user decision: "Photo capture uses auto-context + manual tag -- system auto-fills project/phase/task from current context, tech picks the type (before/during/after/issue) from a quick-select bar."
- `uploadProjectPhoto(token, data)` — Upload to Supabase Storage under `projects/{projectId}/photos/`. Insert project_photos record with project_id, phase_id, task_id (auto-filled from context), tag (selected by tech). Thumbnail generation.

**Issue flagging (PROJ-51):**
Per user decision: "Issue flags create an alert for office -- tech's notes/photos go to office as an alert; office decides whether to create a change order."
- `flagIssue(token, data)` — Insert into project_issue_flags with title, description, severity, photo_urls, project_id, phase_id, task_id. Create an alert in the alerts table with alert_type='project_issue' and link to the issue flag. This does NOT auto-create a change order (per user decision).

**Material usage (PROJ-32, 49):**
- `logMaterialUsage(token, data)` — Insert project_material_usage: material_id (selected from project material list), quantity_used, phase_id, notes. Update project_materials.quantity_used (cumulative).

**Phase completion (PROJ-54):**
- `completePhase(token, phaseId)` — Validate all required tasks are completed. Require at least one completion photo. Set phase status='complete', actual_end_date. Add to project activity_log. Notify office. Trigger milestone invoice check (if a payment milestone is tied to this phase -- call onPhaseComplete from research pattern).

**Equipment tracking (PROJ-55):**
- `assignEquipment(token, projectId, data)` — Insert project_equipment_assignments: equipment_description, assigned_date, notes.
- `returnEquipment(token, assignmentId)` — Set returned_date.

**Route suggestion (PROJ-56):**
- `suggestProjectInRoute(token, techId, date)` — Check if tech has a project phase scheduled for this date. If yes, suggest optimal position in route based on project location vs route stops. Return suggestion (not auto-apply -- office approves).
