**src/actions/projects-change-orders.ts:**

- `createChangeOrder(token, projectId, data)` — Insert into project_change_orders: description, reason (scope_change/unforeseen/customer_request/design_change/regulatory/other), line_items JSONB (each: description, category, quantity, unit_price, total), cost_impact (sum of line items -- positive=increase, negative=decrease), schedule_impact_days, cost_allocation (add_to_final/spread_remaining/collect_immediately per user decision). Auto-generate change_order_number: "CO-XXX" sequential per project. Status='draft'. Add to project activity_log.

- `sendChangeOrder(token, changeOrderId)` — Generate JWT token via signChangeOrderToken (from change-order-token.ts). Send email to customer with change order summary and approval link. Update status='pending_approval'. Add to activity_log.

- `getChangeOrderPublicData(changeOrderId)` — Uses adminDb. Fetch change order with project info, company branding. Returns data for the public approval page.

- `approveChangeOrder(changeOrderId, signatureData)` — Uses adminDb. Set status='approved', approved_at, approved_signature. Then auto-update project (PROJ-59):
  1. Update project.contract_amount += cost_impact
  2. If CO has material line items, add to project_materials
  3. If schedule_impact_days > 0, shift remaining phases forward by that many days (call cascadeDependencies)
  4. Update payment schedule based on cost_allocation:
     - 'add_to_final': increase final milestone amount by cost_impact
     - 'spread_remaining': distribute cost_impact evenly across unpaid milestones
     - 'collect_immediately': create new payment milestone with full CO amount, status='pending'
  5. Add to activity_log noting all changes made

- `declineChangeOrder(changeOrderId, reason)` — Uses adminDb. Set status='declined'. Notify office.

- `getChangeOrderImpact(token, projectId)` — Compute cumulative impact (PROJ-60): original contract + sum(approved COs cost_impact) = current contract. List each CO with its impact. Show on project financial dashboard.

- `convertIssueFlagToChangeOrder(token, issueFlagId)` — Pre-populate change order from issue flag data: title -> description, photos -> reference, phase context. Set issue_flag_id on the change order. Update issue flag status='converted_to_co'. PROJ-61.
