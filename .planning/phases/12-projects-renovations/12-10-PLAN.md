**src/actions/projects-subcontractors.ts:**

**Directory (PROJ-34):**
- `getSubcontractors(token)` — Fetch all subs for org with active flag. Claude's discretion on field structure: name, trade (plumbing/electrical/excavation/decking/masonry/painting/landscaping/other), contact_name, email, phone, address, insurance_cert_path, insurance_expiry, license_number, license_expiry, payment_terms, notes, is_active.
- `createSubcontractor(token, data)` — Insert sub. Upload insurance cert to Supabase Storage.
- `updateSubcontractor(token, id, data)` — Update sub fields.
- `deactivateSubcontractor(token, id)` — Set is_active=false.
- `checkInsuranceExpiry()` — Scheduled check: subs with insurance_expiry within 30 days create alert.

**Assignments (PROJ-35, 36):**
- `assignSubToPhase(token, phaseId, data)` — Create project_phase_subcontractors record: subcontractor_id, scope_of_work, agreed_price, scheduled_start, scheduled_end. Status defaults to 'not_started'. Add to project activity_log.
- `updateSubAssignment(token, assignmentId, data)` — Update status (not_started -> in_progress -> complete / needs_rework), payment fields. Add to activity_log.
- `getSubAssignmentsForProject(token, projectId)` — Fetch all sub assignments across project phases.

**Payments (PROJ-37):**
- `recordSubPayment(token, assignmentId, data)` — Update amount_paid, payment_status. Upload lien waiver document. Track payment as: unpaid -> partial -> paid.
- `getSubPaymentSummary(token, projectId)` — Total owed, total paid, outstanding by sub. Used in project financials.

**Notifications (PROJ-38):**
- `sendSubNotification(token, assignmentId)` — Email to sub's email address with: project address, scope of work, scheduled dates, site access info (from project.site_notes), any special instructions. Use existing Resend/React Email infrastructure. Per research: email only, no portal access for subs.
