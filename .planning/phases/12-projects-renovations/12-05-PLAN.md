  <name>Task 1: Proposal server actions and data layer</name>
  <files>
    src/actions/projects-proposals.ts
  </files>
  <action>
Create proposal-specific server actions:

- `createProposal(token, projectId)` — Create draft proposal for project. If survey data exists, pre-populate scope and measurements. If template has tier_config, pre-populate tiers. Set version=1, status='draft'. Return proposal with empty tiers/addons/milestones for the builder to populate.
- `updateProposal(token, proposalId, data)` — Update proposal fields: scope_description, terms_and_conditions, warranty_info, cancellation_policy, pricing_method, show_line_item_detail, total_amount. Recalculate total from tiers/addons.
- `getProposal(token, proposalId)` — Fetch proposal with tiers, line items, addons, payment milestones. Use LEFT JOIN pattern.
- `getProposalForProject(token, projectId)` — Get active (non-superseded) proposal for a project.

**Tier management:**
- `createProposalTier(token, proposalId, data)` — Create tier (good/better/best) with name, description, price, features array, photo_urls.
- `updateProposalTier(token, tierId, data)` — Update tier fields.
- `deleteProposalTier(token, tierId)` — Remove tier.

**Line items:**
- `addProposalLineItem(token, proposalId, data)` — Add line item with category, description, quantity, unit_price, markup_pct, tier_id (nullable). Compute total = quantity * unit_price * (1 + markup_pct/100). Following the pattern from MEMORY.md: server actions that mutate list data must return fresh state.
- `updateProposalLineItem(token, itemId, data)` — Update and return fresh list.
- `removeProposalLineItem(token, itemId)` — Remove and return fresh list.

**Add-ons:**
- `addProposalAddon(token, proposalId, data)` — Add addon with name, description, price.
- `updateProposalAddon(token, addonId, data)` — Update addon.
- `removeProposalAddon(token, addonId)` — Remove addon.

**Payment schedule:**
- `setPaymentSchedule(token, proposalId, milestones[])` — Bulk upsert payment milestones. Each: name, trigger_phase_id (nullable), percentage, amount, sort_order. Validate percentages sum to 100%.
- `getDefaultPaymentSchedule(token, projectId)` — Return default from template if available.

**Versioning (PROJ-16):**
- `createNewProposalVersion(token, proposalId)` — Set current proposal status='superseded'. Create new proposal copying all tiers/addons/line items/milestones with version=prev+1, status='draft'. Return new proposal.
