  <files>
    src/actions/projects-permits.ts

  <action>
Create permit-specific server actions:

- `createPermit(token, projectId, data)` — Insert into project_permits: permit_type, status='not_applied', fee, notes. Add to project activity_log.
- `updatePermit(token, permitId, data)` — Update permit fields (status, permit_number, dates, inspector info, documents). When status changes to 'approved', add to activity_log. Upload permit documents to Supabase Storage under `projects/{projectId}/permits/`.
- `getPermitsForProject(token, projectId)` — Fetch all permits with status.
- `checkPermitGate(token, projectId)` — Check if project can advance to 'in_progress' stage. Query project template's permit_requirements. If any required permit type does not have status='approved', return { canAdvance: false, blockers: [{permitType, currentStatus}] }. This gate is checked in updateProjectStage before allowing transition to 'in_progress' (PROJ-25).
- `checkPermitExpirations()` — Scheduled job (can be called by a cron edge function or manual trigger). Query all permits where expiration_date is within 30 days AND status='approved' AND no recent alert exists. Create alert with alert_type='permit_expiring' (PROJ-26). Use adminDb for this system operation.
- `uploadHoaDocument(token, projectId, file)` — Upload to Supabase Storage under `projects/{projectId}/hoa/`. Insert into project_documents with document_type='hoa'. PROJ-27.
- `getProjectDocuments(token, projectId)` — Fetch all project_documents ordered by type and created_at.
- `deleteProjectDocument(token, docId)` — Soft-archive: set archived_at (per PROJ-91, no hard deletes).

Claude's discretion on permit tracking field set:
- Permit types: Building, Electrical, Plumbing, Mechanical, Demolition, Excavation, Pool/Spa (varies by jurisdiction)
- Inspector fields: name, phone, email
- Status lifecycle: not_applied -> applied -> under_review -> approved/denied -> (expired after date)
- Documents: uploaded PDFs/images of permit applications, approvals, inspection reports
  </action>

Run `npx tsc --noEmit`. Test permit CRUD, gate check logic (blocks advancement without approved permits), and document upload to Supabase Storage.
  </verify>
