**src/actions/projects.ts — Core server actions:**
- `getProjects(token)` — fetch all projects for org with customer name, phase count, days in stage (computed from stage_entered_at). Use LEFT JOIN pattern (not correlated subqueries) per RLS pitfalls.
- `createProject(token, data)` — create project with lead_source, customer_id, project_type, name. Auto-generate project_number as "PRJ-XXXX" (sequential per org). Set stage='lead', status='active'. If template_id provided, populate project_phases and project_phase_tasks from template default_phases.
- `updateProjectStage(token, projectId, newStage)` — update stage + stage_entered_at. Add entry to activity_log JSONB array. Update last_activity_at.
- `holdProject(token, projectId, reason)` — set status='on_hold', on_hold_reason. Add to activity_log.
- `resumeProject(token, projectId)` — set status='active', clear on_hold_reason. Add to activity_log.
- `getProjectTemplates(token)` — fetch all templates for org.
- `createProjectTemplate(token, data)` — create template with name, project_type, default_phases JSONB, tier_config JSONB, default_payment_schedule JSONB.
- `updateProjectTemplate(token, id, data)` — update template fields.
- `deleteProjectTemplate(token, id)` — soft-delete (set is_active=false).
- `getProjectPipelineMetrics(token)` — return stage counts, avg days per stage, lead-to-close conversion rate (PROJ-83).
