**src/actions/projects-materials.ts:**
- `populateMaterialsFromProposal(token, projectId)` — When proposal is approved, copy all material-category line items from approved proposal into project_materials table. Set quantity_estimated from line item quantity, unit_cost_estimated from line item unit_price. This links PROJ-28 (material list from approved proposal).
- `getMaterials(token, projectId)` — Fetch all project materials with computed fields: total_estimated, total_actual, variance (actual - estimated), variance_pct. Use LEFT JOIN pattern.
- `addMaterial(token, projectId, data)` — Manually add material (for items not from proposal). Return fresh list.
- `updateMaterial(token, materialId, data)` — Update material fields. Return fresh list.

**Purchase Orders:**
- `createPurchaseOrder(token, projectId, data)` — Create PO from selected materials grouped by supplier. Generate PO number (sequential per org: "PO-XXXX"). Create PO line items linking to project_materials. Update material order_status to 'ordered'. Add to project activity_log.
- `getPurchaseOrders(token, projectId)` — Fetch POs with line items.
- `updatePurchaseOrderStatus(token, poId, status)` — Update PO status.
- `getPurchaseOrderPdf(token, poId)` — Generate PO PDF.

**Receiving:**
- `receiveMaterial(token, materialId, data)` — Create receipt record: quantity_received, received_by, photo_url (optional delivery photo). Update material.quantity_received (cumulative). If quantity_received < quantity_ordered, set order_status='partial'. If equal, set 'received'. Update material.unit_cost_actual if actual price differs from estimate.
- `getReceipts(token, projectId)` — Fetch all receipts.

**Returns:**
- `returnMaterial(token, materialId, data)` — Create return record: quantity_returned, return_reason, credit_amount. Update material quantities. PROJ-33.

**Cost Variance:**
- `getMaterialCostVariance(token, projectId)` — Compute per-material and total variance: sum(quantity_used * unit_cost_actual) vs sum(quantity_estimated * unit_cost_estimated). Create alert if total actual exceeds estimate by configurable threshold (PROJ-31).

6. Subtotal
7. Notes section
8. Authorized signature line

HEX colors only. Match quote-pdf.tsx styling conventions.
