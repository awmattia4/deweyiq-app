  <name>Task 1: Survey scheduling and server actions</name>
  <files>
    src/actions/projects-survey.ts
  </files>
  <action>
Create survey-specific server actions:

- `scheduleSurvey(token, projectId, data)` — Creates a route_stop with stop_type='survey' and project_id FK. Data includes: tech_id (assigned surveyor), date, time window. Updates project stage to 'site_survey_scheduled'. Adds activity_log entry. The route_stop should include the customer address and the project_id so the tech's route shows it as a survey stop.

  NOTE: This requires adding `stop_type` column to route_stops if not already present (check existing schema -- it may already support 'service'|'work_order' types; add 'survey' to the set). Also add `project_id` FK to route_stops if not present.

- `completeSurvey(token, projectId, surveyData)` — Insert into project_surveys table with: measurements (JSONB: pool_dimensions, deck_area, equipment_pad_size, plumbing_runs, electrical_capacity, etc.), existing_conditions (JSONB: surface_condition, equipment_condition, structural_issues, drainage, etc.), access_constraints, utility_locations, hoa_requirements, notes. Upload survey photos to Supabase Storage under `projects/{projectId}/survey/` path. Update project stage to 'survey_complete'. Add activity_log entry.

- `getSurveyData(token, projectId)` — Fetch survey data for proposal builder pre-population. Returns measurements, conditions, photo URLs, surveyor notes.

- `getSurveyChecklist()` — Returns default survey checklist items (Claude's discretion on defaults):
  1. Pool/spa dimensions (length, width, depth)
  2. Deck measurements and condition
  3. Equipment pad location and size
  4. Existing equipment inventory
  5. Plumbing access points
  6. Electrical panel capacity
  7. Gas line location (if applicable)
  8. Drainage assessment
  9. Structural condition (cracks, settling, heaving)
  10. HOA restrictions (fence height, material, color)
  11. Neighbor notification requirements
  12. Utility marking status (811 call)
  13. Access for heavy equipment (gate width, overhead clearance)
  14. Soil conditions (if excavation needed)
  15. Photos: front approach, pool overview, equipment area, deck, any issues
  </action>

Run `npx tsc --noEmit`. Test scheduleSurvey creates a route_stop with correct type. Test completeSurvey stores survey data.
  </verify>
