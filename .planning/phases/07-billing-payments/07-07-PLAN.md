    **Create `src/actions/reports.ts`** — "use server" module with report queries:

    1. `getArAging()`:
       - Uses withRls. Owner+office.
       - SQL query bucketing unpaid invoices into aging categories (per Research Pattern 6):
         - "Current" (not yet past due)
         - "1-30 days" past due
         - "31-60 days" past due
         - "61-90 days" past due
         - "90+ days" past due
       - Query invoices with status IN ('sent', 'overdue') AND paid_at IS NULL.
       - Calculate days overdue from due_date (not issued_at — use due_date for proper aging).
       - Group by customer: for each customer, show total per bucket + grand total.
       - Return: { customers: ArAgingCustomer[], totals: { current, d1_30, d31_60, d61_90, d90_plus, total } }.
       - Use LEFT JOIN with customers table (not correlated subquery per MEMORY.md).

    2. `getRevenueByCustomer(startDate: string, endDate: string)`:
       - Uses withRls. Owner+office.
       - Query invoices with status='paid' and paid_at within the date range.
       - Group by customer: sum of total per customer.
       - Include invoice count per customer.
       - Sort by total descending.
       - Return: { customers: { id, name, totalRevenue, invoiceCount }[], grandTotal }.

    3. `getPnlReport(startDate: string, endDate: string)`:
       - Uses withRls. Owner+office.
       - **Revenue**: Sum of all paid invoices in date range.
       - **Expenses**: Query the `expenses` table for the date range. Group by category for breakdown. Sum for total expenses.
       - Revenue breakdown: by billing model (per_stop, flat_rate, plus_chemicals, custom).
       - Revenue breakdown: by month within the date range.
       - Expense breakdown: by category (chemicals, fuel, equipment, labor, etc.).
       - Expense breakdown: by month within the date range.
       - Return: { revenue: { total, byModel, byMonth }, expenses: { total, byCategory, byMonth }, netIncome }.
       - NOTE: Bank reconciliation is explicitly out of scope for Phase 7 — it requires Plaid integration planned for Phase 11 (ACCT-06, ACCT-07). The P&L shows manual expense entries only.

    4. `exportFinancialCsv(type: 'invoices' | 'payments' | 'ar_aging' | 'expenses', startDate?: string, endDate?: string)`:
       - Uses withRls. Owner only.
       - For 'invoices': Export all invoices in date range with: invoice_number, customer_name, issued_at, due_date, total, tax, status, paid_at, payment_method, billing_model, billing_period.
       - For 'payments': Export all settled payment_records in date range with: invoice_number, customer_name, amount, method, settled_at, stripe_payment_intent_id.
       - For 'ar_aging': Export AR aging snapshot with: customer_name, invoice_number, total, due_date, days_overdue, aging_bucket.
       - For 'expenses': Export all expenses in date range with: date, category, amount, description, created_by.
       - Generate CSV string with headers and rows.
       - Return the CSV string (client handles download via Blob URL).
