    **Create `src/lib/qbo/client.ts`** — QBO client factory with token refresh:
    - Import `OAuthClient` from `intuit-oauth` and `QuickBooks` from `node-quickbooks`.
    - `getQboClient(orgId: string)`:
      - Load org_settings for qbo_realm_id, qbo_access_token, qbo_refresh_token, qbo_token_expires_at.
      - If not connected (no qbo_realm_id), throw Error("QBO not connected").
      - If token expires within 60 seconds:
        - Use Postgres advisory lock to prevent concurrent refresh (per Research Pitfall 2):
          ```sql
          SELECT pg_advisory_xact_lock(hashtext('qbo_refresh_' || org_id))
          ```
        - Re-read org_settings inside the lock (another request may have already refreshed).
        - If still expired, call `oauthClient.refreshUsingToken(refreshToken)`.
        - ALWAYS store the NEW refresh token returned (old one is immediately invalid).
        - Update org_settings with new access_token, refresh_token, token_expires_at.
      - Return configured QuickBooks client instance.
    - `isQboConnected(orgId: string)`: Check org_settings.qbo_connected.
    - Wrap QuickBooks callback methods with Promise wrappers for async/await usage:
      ```typescript
      export function qboPromise<T>(fn: (cb: (err: any, result: T) => void) => void): Promise<T> {
        return new Promise((resolve, reject) => fn((err, result) => err ? reject(err) : resolve(result)))
      }
      ```

    **Create `src/lib/qbo/mappers.ts`** — entity mapping functions:

    1. `mapCustomerToQbo(customer: PoolCoCustomer)` -> QBO Customer object:
       - PrimaryEmailAddr: { Address: customer.email }
       - DisplayName: customer.full_name
       - PrimaryPhone: { FreeFormNumber: customer.phone }
       - BillAddr: { Line1: customer.address, City, PostalCode } (parse from address string or use separate fields)
       - If customer has qbo_customer_id, include Id and SyncToken for update.

    2. `mapInvoiceToQbo(invoice: PoolCoInvoice, lineItems: LineItem[], qboCustomerId: string)` -> QBO Invoice object:
       - CustomerRef: { value: qboCustomerId }
       - TxnDate: invoice.issued_at
       - DueDate: invoice.due_date
       - Line: lineItems mapped to QBO format:
         - DetailType: "SalesItemLineDetail"
         - Amount: lineItem.total
         - Description: lineItem.description
         - SalesItemLineDetail: { Qty: lineItem.quantity, UnitPrice: lineItem.unit_price }
       - If invoice has qbo_invoice_id, include Id and SyncToken for update.

    3. `mapPaymentToQbo(payment: PaymentRecord, qboInvoiceId: string, qboCustomerId: string)` -> QBO Payment object:
       - CustomerRef: { value: qboCustomerId }
       - TotalAmt: payment.amount
       - TxnDate: payment.settled_at
       - Line: [{ Amount: payment.amount, LinkedTxn: [{ TxnId: qboInvoiceId, TxnType: "Invoice" }] }]
       - PaymentMethodRef based on payment.method (map 'card' to 'Credit Card', 'ach' to 'Check', etc.)

    4. `mapQboPaymentToPoolCo(qboPayment: any)` -> partial PaymentRecord:
       - Extract amount, method, and linked invoice ID from QBO payment object.
       - Used for inbound webhook processing.

    **Create `src/actions/qbo-sync.ts`** — "use server" sync actions:

    1. `syncCustomerToQbo(customerId: string)`:
       - Fetch customer from DB.
       - Check if org has QBO connected and payment_provider includes 'qbo' or 'both'.
       - If not connected or QBO not in payment stack, silently return (no-op).
       - Get QBO client via getQboClient.
       - If customer has qbo_customer_id:
         - Fetch existing QBO customer to get SyncToken.
         - Update via `qbo.updateCustomer(mapCustomerToQbo(customer))`.
       - If no qbo_customer_id:
         - Create via `qbo.createCustomer(mapCustomerToQbo(customer))`.
         - Save returned QBO Customer ID to customer.qbo_customer_id via adminDb.
       - Update org_settings.qbo_last_sync_at.
       - Wrap in try/catch — QBO sync failure should NOT block the primary operation. Log error, create alert.

    2. `syncInvoiceToQbo(invoiceId: string)`:
       - Fetch invoice with line items and customer.
       - If customer has no qbo_customer_id, sync customer first via syncCustomerToQbo.
       - If invoice has qbo_invoice_id:
         - Fetch existing QBO invoice for SyncToken, then update.
       - If no qbo_invoice_id:
         - Create via `qbo.createInvoice(mapInvoiceToQbo(...))`.
         - Save returned QBO Invoice ID to invoice.qbo_invoice_id via adminDb.
       - Update qbo_last_sync_at.
       - Silently fail on error (log + alert, don't block).

    3. `syncPaymentToQbo(paymentId: string)`:
       - Fetch payment_record, associated invoice, customer.
       - If invoice has no qbo_invoice_id, sync invoice first.
       - If customer has no qbo_customer_id, sync customer first.
       - Create QBO Payment via `qbo.createPayment(mapPaymentToQbo(...))`.
       - Save qbo_payment_id on payment_record.
       - Update qbo_last_sync_at.
       - Silently fail on error.

    4. `handleQboWebhook(payload: any)`:
       - Parse QBO webhook notification.
       - For each entity change:
         - If entity is "Payment" and operation is "Create" or "Update":
           - Fetch the QBO payment by ID.
           - Find the linked invoice via QBO invoice ID.
           - Find the PoolCo invoice by qbo_invoice_id.
           - If PoolCo invoice found and no existing payment_record for this qbo_payment_id:
             - Create payment_record with method='qbo', status='settled', qbo_payment_id.
             - Update invoice status to 'paid' if fully paid.
         - If entity is "Customer" — skip (PoolCo is source of truth per locked decision).
         - If entity is "Invoice" — skip (PoolCo is source of truth).
       - Use adminDb (webhook has no user session).

    5. `disconnectQbo()`:
       - Uses withRls. Owner only.
       - Clear all QBO fields in org_settings: set qbo_connected=false, null out tokens, realm_id.
       - If payment_provider was 'qbo', set to 'none'. If 'both', set to 'stripe'.
       - Revalidate paths.

    6. `getQboStatus()`:
       - Fetch org_settings QBO fields.
       - Return: { connected, realmId, lastSyncAt, companyName (if available) }.
