    **Create `src/lib/pay-token.ts`** — JWT token system for public payment pages:
    - Mirror `src/lib/quotes/quote-token.ts` exactly in structure.
    - Payload: `{ invoiceId: string }`
    - Secret: `INVOICE_TOKEN_SECRET` env var (separate from QUOTE_TOKEN_SECRET and REPORT_TOKEN_SECRET).
    - Algorithm: HS256
    - Expiry: 365 days (longer than invoice lifecycle — invoice can stay outstanding for months)
    - Export `signPayToken(invoiceId: string)` and `verifyPayToken(token: string)`

    **Create `src/lib/emails/invoice-email.tsx`** — React Email invoice delivery template:
    - Follow the exact same pattern as `src/lib/emails/quote-email.tsx` (dark-themed, branded).
    - Header: Company name (or logo if available).
    - Body: Invoice number, total amount (bold, large), billing period ("Service period: Mar 1 - Mar 31"), due date, customer name.
    - If billing_model is 'per_stop', show number of stops. If 'flat_rate', show "Monthly Service".
    - CTA button: "Pay Now" linking to `{APP_URL}/pay/{token}` — the branded payment page (built in Plan 04).
    - Footer: "This invoice was generated by {company name}. If you have questions, reply to this email."
    - All colors hex only (React Email + Resend compatibility).

    **Update `src/actions/invoices.ts`** — enhance `sendInvoice(invoiceId: string, options: { email?: boolean, sms?: boolean })`:
    - Finalize the invoice if still draft (assign number via existing adminDb counter pattern).
    - Set status to 'sent', issued_at to now.
    - Generate pay token via `signPayToken(invoiceId)`.
    - Build the payment URL: `${process.env.NEXT_PUBLIC_APP_URL}/pay/${token}`.
    - If `options.email` (default true):
      - Fetch customer email, org name, org logo, invoice details.
      - Generate invoice PDF buffer via existing `renderToBuffer(createElement(InvoiceDocument, props) as any)` pattern.
      - Render InvoiceEmail to HTML via `render(createElement(InvoiceEmail, props))`.
      - Send via Resend SDK with PDF attachment (base64 encoded), same pattern as quote email.
      - Update invoices.sent_at to now.
    - If `options.sms` and customer has phone number:
      - Invoke Supabase Edge Function `send-invoice-sms` with phone, paymentUrl, invoiceNumber, total, companyName.
      - Update invoices.sent_sms_at to now.
    - If neither email nor sms specified, default to email only.
    - Revalidate paths.

    **Add `sendAllInvoices(invoiceIds: string[])` action** — batch send:
    - Iterate over invoice IDs sequentially (not Promise.all — each one sends email+generates PDF).
    - Call sendInvoice for each with both email and SMS enabled.
    - Return { sent: number, failed: number, errors: string[] }.
    - This is the "send all" action for the batch invoicing flow.

    **Update `src/components/work-orders/invoice-list.tsx`** — add:
    - "Send" button on each draft/unsent invoice row.
    - "Send All" button at the top when viewing draft invoices (for batch flow).
    - Send options: dropdown with "Email Only", "SMS Only", "Email + SMS". SMS options only visible if customer has phone number.
    - Loading states during send operations.

    - Note: This single Edge Function handles both invoice and quote SMS to avoid creating duplicate functions.
