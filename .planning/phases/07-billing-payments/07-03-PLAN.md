  <name>Task 1: Stripe singleton, Connect onboarding API routes, and account status check</name>
  <files>
    src/lib/stripe/client.ts
    src/app/api/connect/stripe/onboard/route.ts
    src/app/api/connect/stripe/return/route.ts

  <action>
    **Create `src/lib/stripe/client.ts`** — Stripe server-side singleton:
    ```typescript
    import Stripe from "stripe"

    export const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
      apiVersion: "2025-02-24.acacia",  // or latest stable
      typescript: true,
    })
    ```
    This is the platform Stripe instance (not per-connected-account). It's used for all Connect operations.

    **Create `src/app/api/connect/stripe/onboard/route.ts`** — POST handler:
    - Authenticate via `getCurrentUser()` — owner only.
    - Check if org already has a `stripe_account_id` in org_settings:
      - If not: create a Standard connected account via `stripe.accounts.create({ type: "standard" })`. Save the `acct_*` ID to org_settings.stripe_account_id using adminDb (org_settings UPDATE RLS is owner-only but use adminDb for consistency with Phase 6 pattern).
      - If yes: use existing account ID.
    - Create an Account Link via `stripe.accountLinks.create()`:
      - `account`: the acct_* ID
      - `refresh_url`: `${NEXT_PUBLIC_APP_URL}/settings?stripe=refresh` (user re-clicks if link expires)
      - `return_url`: `${NEXT_PUBLIC_APP_URL}/settings?stripe=return` (redirect after onboarding)
      - `type`: `"account_onboarding"`
    - Return `{ url: accountLink.url }` — client redirects to this Stripe-hosted URL.

    **Create `src/app/api/connect/stripe/return/route.ts`** — GET handler (return URL target):
    - This is where Stripe redirects after onboarding completion or partial completion.
    - Authenticate user, fetch the org's stripe_account_id.
    - Call `stripe.accounts.retrieve(accountId)` to check `charges_enabled` and `details_submitted`.
    - If `charges_enabled === true && details_submitted === true`:
      - Update org_settings: `stripe_onboarding_done = true`
      - If `payment_provider` is 'none', set to 'stripe'.
    - Redirect to `/settings?stripe=success` or `/settings?stripe=incomplete` based on status.
    - This route exists as a redirect handler, not a UI page.

    **Create a server action `getStripeAccountStatus()`** in a new `src/actions/stripe-connect.ts`:
    - Fetch the org's stripe_account_id from org_settings.
    - If no account, return `{ connected: false }`.
    - If account exists, call `stripe.accounts.retrieve(accountId)`.
    - Return: `{ connected: boolean, chargesEnabled: boolean, payoutsEnabled: boolean, email: string | null, businessName: string | null, onboardingComplete: boolean }`.
    - Use adminDb to read org_settings (consistent with other settings reads).
  </action>
  <verify>
    `npx tsc --noEmit` passes.
    `npm run build` succeeds.
    API route files exist at correct paths.
    Stripe import resolves (package installed in Plan 01).
  </verify>
  <done>
    Stripe singleton created for server-side use.
    Connect onboarding POST route creates account and returns account link URL.
    Return route checks onboarding status and updates org_settings.
    Server action provides Stripe account status for UI display.
  </done>
