    **Create `src/actions/billing.ts`** — "use server" module with billing model logic:

    1. `updateCustomerBillingModel(customerId: string, model: string, flatRateAmount?: string)` — updates customer.billing_model and optionally customer.flat_rate_amount. Uses withRls. Revalidates /customers/[id].

    2. `generateInvoiceForCustomer(customerId: string, periodStart: string, periodEnd: string)` — creates a draft invoice for one customer based on their billing model:
       - **per_stop**: Query route_stops with status='completed' in date range for this customer. Create one line item per stop with stop_date and description like "Pool Service - {date}". Use the org's default hourly rate or a per-stop price from org_settings. Each line item should reference the visit_id if available.
       - **flat_rate**: Single line item with the customer's flat_rate_amount and description "Monthly Service - {period}".
       - **plus_chemicals**: Start with per-stop line items, then query service_visits in the date range for chemistry_readings.dosing data. For each chemical dosed, add a line item with item_type='chemical', description like "{chemical} - {date}", quantity from dose amount. Handle null chemistry_readings gracefully (skip, don't error). Office edits before finalizing.
       - **custom**: Create a draft invoice with no line items — office adds them manually.
       - All models: Set billing_model, billing_period_start, billing_period_end on the invoice. Calculate due_date = invoice created_at + org_settings.default_payment_terms_days.
       - Use withRls for data queries, adminDb for the invoice counter increment (same pattern as Phase 6 finalizeInvoice).
       - Return the created invoice ID.

    3. `generateAllInvoices(periodStart: string, periodEnd: string)` — bulk invoice generation:
       - Query all active customers with a non-null billing_model.
       - For each customer, check if an invoice already exists for this exact period (prevent duplicates).
       - Generate invoices sequentially (NOT Promise.all — per research Pitfall 6, concurrent invoice number generation causes race conditions).
       - Use a single adminDb transaction to reserve N invoice numbers at once: `UPDATE org_settings SET next_invoice_number = next_invoice_number + {count} WHERE org_id = $1 RETURNING next_invoice_number - {count} AS start_number`. Then assign sequential numbers.
       - Return { created: number, skipped: number, errors: string[] }.
       - Revalidate /work-orders (where invoice list lives).

    4. `getPlusChemicalsLineItems(customerId: string, periodStart: string, periodEnd: string)` — extracts chemical dosing from service_visits.chemistry_readings JSONB for the period. Returns DraftLineItem[] with chemical name, date, quantity, unit. Handles null readings gracefully. Used by generateInvoiceForCustomer for plus_chemicals model.
