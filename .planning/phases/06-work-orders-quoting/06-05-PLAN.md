**Create src/actions/quotes.ts** — "use server" directive. Implement:

1. `createQuote(workOrderId)` — Creates a new quote for the WO. Auto-generates quote_number using atomic increment of org_settings.next_quote_number (same pattern as invoice numbering from 06-RESEARCH.md Pattern 4 — `UPDATE org_settings SET next_quote_number = next_quote_number + 1 WHERE org_id = $1`). Copies current WO line items into a snapshot_json. Sets version=1, status='draft'. Sets expires_at from org default_quote_expiry_days. Returns quote id. Uses withRls for the quote insert but adminDb for the atomic counter increment (counter is in org_settings which is owner-only for update — but we need to increment it from office context too, so use adminDb for the counter).

2. `sendQuote(quoteId)` — The core send flow:
   a. Fetch quote + WO + customer + org settings (company name, logo, terms) using adminDb.
   b. Build QuoteDocumentProps from the data.
   c. Call `renderToBuffer(<QuoteDocument {...props} />)` to generate PDF buffer.
   d. Sign approval token: `signQuoteToken(quoteId)`.
   e. Build approval URL: `${process.env.NEXT_PUBLIC_APP_URL}/quote/${token}` (public page route, not API route).
   f. Render email HTML: `render(QuoteEmailTemplate({...}))` from @react-email/render.
   g. Send via Resend SDK DIRECTLY (NOT Edge Function — per 06-RESEARCH.md Pitfall 4):
   ```
   resend.emails.send({
     from: `${companyName} <quotes@poolco.app>`,
     to: [customerEmail],
     subject: `Quote #${quoteNumber} from ${companyName}`,
     html: emailHtml,
     attachments: [{ filename: `quote-${quoteNumber}.pdf`, content: Buffer.from(pdfBuffer).toString('base64') }]
   })
   ```
   h. Update quote: status='sent', sent_at=now().
   i. Update WO status to 'quoted' (calls updateWorkOrderStatus).
   j. Append activity_log event: 'quote_sent'.
   k. revalidatePath.

3. `reviseQuote(quoteId)` — Creates a new version of a sent/declined/changes_requested quote:
   a. Snapshot current quote state into snapshot_json.
   b. Increment version.
   c. Reset status to 'draft'.
   d. Office can edit line items on the WO, then re-send.
   e. revalidatePath.

4. `extendQuote(quoteId, newExpirationDate)` — Updates expires_at on an expired quote. Resets status to 'sent' if it was 'expired'. revalidatePath.

5. `getQuotesForWorkOrder(workOrderId)` — Fetches all quotes for a WO, ordered by version desc. Uses withRls.
