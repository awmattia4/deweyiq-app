  <action>
**Create src/components/work-orders/line-item-editor.tsx** — "use client" component:
- Props: workOrderId, lineItems (initial array), orgSettings (for defaults: default_hourly_rate, default_parts_markup_pct, default_tax_rate), editable (boolean — false when WO is invoiced), onLineItemsChange (callback for parent re-render).

**Line items table:**
- Each row shows: drag handle (for reorder), description, type badge (Part/Labor/Other), qty, unit, unit price, line total, actions (edit/delete).
- Optional items show a dashed border or "(optional)" label.
- Taxable items show a small tax indicator.

**Add line item flow:**
- "Add Item" button opens an inline form or popover with:
  - **Search catalog:** Type-ahead search field that queries `getCatalogItems(search)`. Results show as a dropdown. Selecting a catalog item auto-fills: description, unit, unit_price (from default_sell_price), unit_cost (from default_cost_price), is_taxable, item_type.
  - **Or custom item:** If no catalog match, user types a custom description.
  - **Item type:** Toggle: Part / Labor / Other.
  - **For labor items:** Labor type toggle: Hourly / Flat Rate. If hourly, shows rate field (pre-filled from org default_hourly_rate) + estimated hours.
  - **For parts:** Unit price auto-applies markup from org default_parts_markup_pct over unit_cost. Office can override the sell price.
  - **Quantity + Unit:** Numeric input + unit selector (each, hour, foot, gallon, bag, etc.).
  - **Per-item discount (optional):** Toggle to show discount fields: type (percent/fixed) + value + reason.
  - **Taxable toggle:** Checkbox, default based on item_type (parts: true, labor: false — per locked decision).
  - **Optional toggle:** Checkbox for marking line item as optional (customer can include/exclude on quote approval).
  - **"Save to catalog" checkbox:** When adding a custom item, option to save it to the parts catalog for future use. Calls `addCatalogItem`.

- On save: calls server action to persist the line item (e.g., `addLineItemToWorkOrder` or similar action that inserts into work_order_line_items). Router refresh to update page.

**Edit line item:**
- Click edit icon on a row → inline edit or popover with same fields as add.
- Save calls update action.

**Delete line item:**
- Confirmation then calls delete action.

**Reorder:**
- Drag-and-drop to reorder line items (update sort_order). Can use a simple up/down button approach if DnD is too heavy for this context.

**Totals section:**
- Below the items table, show:
  - Subtotal (sum of line totals for non-optional + included optional items)
  - Discount (if WO has whole-order discount)
  - Tax (sum of taxable line totals * org tax rate, unless WO or customer is tax_exempt)
  - **Total** (subtotal - discount + tax)
- All calculations use `toFixed(2)` for display only. Canonical totals are server-computed.

**Add missing server actions to work-orders.ts** (if not already from Plan 01):
- `addLineItemToWorkOrder(woId, data)` — Insert line item.
- `updateLineItem(lineItemId, data)` — Update line item fields.
- `deleteLineItem(lineItemId)` — Delete line item.
- `reorderLineItems(woId, orderedIds)` — Update sort_order for each item.

**CRITICAL:** Use local `useState<string>` for decimal number inputs (unit_price, quantity, discount_value, hours) to avoid the parseFloat("7.") pitfall per MEMORY.md. Only flush to server when value is a complete number.
