**Create src/actions/invoices.ts** — "use server" directive. Implement:

1. `prepareInvoice(workOrderId)` — Creates a draft invoice from a completed WO:
   - Validates WO is in 'complete' status.
   - Fetches WO line items. If a quote was approved with optional item selections (quote.approved_optional_item_ids), only include items that were approved.
   - Creates an invoice record with status='draft', work_order_ids=[workOrderId].
   - Copies WO line items to invoice_line_items with calculated line_total (quantity * unit_price - per-item discount).
   - Calculates: subtotal (sum of line totals), tax_amount (sum of taxable line totals * org tax rate, unless customer.tax_exempt or WO.tax_exempt), discount_amount (WO-level discount), total.
   - Returns invoice id.
   - Uses withRls.

2. `addWorkOrderToInvoice(invoiceId, workOrderId)` — For multi-WO invoicing:
   - Validates WO is complete and belongs to same customer as the invoice.
   - Appends workOrderId to invoice.work_order_ids.
   - Copies WO line items to invoice_line_items.
   - Recalculates totals.
   - Uses withRls.

3. `updateInvoiceLineItem(lineItemId, data)` — Allows office to adjust quantities, prices, or add/remove items during invoice preparation.

4. `addInvoiceLineItem(invoiceId, data)` — Add a new line item to the invoice.

5. `removeInvoiceLineItem(lineItemId)` — Remove a line item from invoice.

6. `finalizeInvoice(invoiceId)` — Finalizes the invoice:
   - Generates invoice number using atomic counter: `UPDATE org_settings SET next_invoice_number = next_invoice_number + 1 WHERE org_id = $1` via adminDb. Read back the value. Format: `{prefix}-{padded_number}` (e.g., "INV-0042").
   - Sets invoice status='sent' (or 'finalized' — ready for delivery).
   - Updates all referenced WOs to status='invoiced' via updateWorkOrderStatus.
   - Generates invoice PDF (for storage/email — Phase 7 handles delivery).
   - Returns the finalized invoice with number.
   - Uses adminDb for the counter, withRls for the invoice update.

7. `getInvoices(filters?)` — Fetches invoices with LEFT JOIN to customers. Filters by status[], customerId, dateRange. Orders by created_at desc. Uses withRls.

8. `getInvoice(id)` — Fetches single invoice with line items and customer info. Uses withRls.

9. `createCreditNote(invoiceId, reason, adjustmentAmount)` — Creates a credit note as a new invoice record with negative total, referencing the original invoice. Status='sent'. Uses withRls.

**Create src/lib/pdf/invoice-pdf.tsx** — React PDF document component:
- Same @react-pdf/renderer pattern as quote-pdf.tsx. All hex colors.
- Props (InvoiceDocumentProps): invoiceNumber, invoiceDate, companyName, companyLogoUrl, companyAddress (if in org settings), customerName, customerAddress, lineItems array (description, quantity, unit, unitPrice, lineTotal), subtotal, taxRate, taxAmount, discountAmount, total, notes, workOrderNumbers (list of WO titles/refs).
- Layout matching quote PDF style for consistency:
  - Header: company info + "INVOICE #{number}" + date.
  - Bill to: customer name + address.
  - Reference: "Work Order(s): {WO references}".
  - Line items table (same column layout as quote PDF).
  - Totals: subtotal, discount, tax, total.
  - Notes section.
  - Payment terms / instructions (placeholder for Phase 7 to add payment link).

- Authenticated (check getCurrentUser).
- Fetches invoice data via adminDb.
- Renders PDF with `renderToBuffer(<InvoiceDocument {...props} />)`.
- Returns PDF response with Content-Type + Content-Disposition headers.
