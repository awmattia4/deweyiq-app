  <action>
    **`src/lib/accounting/journal.ts`** — Core accounting engine (uses adminDb for system operations):

    1. **`validateEntryBalance(lines: { accountId: string; amount: string }[])`** — Validates that all line amounts sum to zero (within 0.01 tolerance for rounding). Throws if imbalanced. Uses numeric strings, never floats.

    2. **`createJournalEntry(input: { orgId, entryDate, description, sourceType, sourceId, lines: { accountId, amount, description? }[], createdBy? })`** — Validates balance, checks accounting period is not closed (if period exists for the entry_date), inserts journal_entries row + all journal_entry_lines rows in a single transaction. Returns the created entry ID. Uses adminDb (system operation — auto-generated entries aren't user-initiated).

    3. **`reverseJournalEntry(entryId: string, reason: string)`** — Creates a new journal entry with all line amounts negated (debits become credits, credits become debits). Links via reversal_of. Marks original as is_reversed=true. Does NOT delete the original (immutable ledger per ACCT-13 audit trail). Uses adminDb.

    4. **`getJournalEntriesForSource(sourceType: string, sourceId: string)`** — Finds journal entries linked to a specific source (e.g., invoice, payment). Used to check if an entry already exists (idempotency).

    **Auto-generation helper functions** (in same file):
    5. **`createInvoiceJournalEntry(invoiceId: string)`** — Fetches invoice, looks up AR (1100) and Revenue (4000) and Tax Payable (2100) accounts. Creates: Dr AR +total, Cr Revenue -subtotal, Cr Tax Payable -taxAmount. Idempotent: checks if entry already exists for this invoice.

    6. **`createPaymentJournalEntry(paymentId: string)`** — Fetches payment + invoice. If Stripe: Dr Stripe Clearing (1020) +netAmount, Dr Stripe Fees (5600) +feeAmount, Cr AR -grossAmount. If manual/check: Dr Checking (1000) +amount, Cr AR -amount.

    7. **`createExpenseJournalEntry(expenseId: string)`** — Fetches expense. Dr appropriate expense account +amount, Cr Checking (1000) or AP (2000) -amount depending on payment status.

    8. **`createRefundJournalEntry(paymentId: string, refundAmount: string)`** — Reversal pattern: Dr Revenue, Cr Bank/Stripe.

    **Extend `src/lib/accounting/chart-of-accounts.ts`** (from Plan 01 Task 2):
    - Ensure `seedChartOfAccounts` is called lazily on first accounting access (check if chart_of_accounts has rows for the org; if not, seed).
    - Add `ensureChartOfAccounts(orgId: string)` wrapper that seeds only if needed.

    **New file: `src/actions/accounting.ts`**:
    1. `getChartOfAccounts()` — Owner/office only. Returns all accounts for org, ordered by number. Includes running balance (sum of all journal_entry_lines for the account). Calls ensureChartOfAccounts first.
    2. `createAccount(input: { accountNumber, accountName, accountType, displayName, parentId? })` — Owner only. Creates custom account. Cannot duplicate existing account numbers.
    3. `updateAccount(accountId: string, updates)` — Owner only. Cannot update system accounts' type or number. Can update name/display_name.
    4. `deleteAccount(accountId: string)` — Owner only. Cannot delete system accounts. Cannot delete accounts with journal_entry_lines (has_transactions check).
    5. `getJournalEntries(filters: { startDate, endDate, sourceType?, accountId? })` — Owner only. Paginated list of journal entries with lines. For simplified view: show description + total amounts. For accountant mode: show full debit/credit lines.
    6. `createManualJournalEntry(input: { entryDate, description, lines })` — Owner only. Accountant mode only (check org_settings.accountant_mode_enabled). Validates balance. Creates entry with source_type='manual'.
  </action>
