<task type="auto">
  <name>Task 1: Add sales tax, period close, and audit trail actions</name>
  <files>src/actions/accounting.ts</files>
  <action>
    Extend `src/actions/accounting.ts` with:

    **Sales Tax:**
    1. `getSalesTaxRates()` — Owner only. Returns configured tax rates. Use the existing `org_settings.default_tax_rate` as the base. For multi-jurisdiction: store additional rates in a JSONB column on org_settings (e.g., `sales_tax_rates: [{ jurisdiction: "County A", rate: 0.07 }, ...]`). If no custom rates, use the single default rate.
    2. `updateSalesTaxRates(rates: { jurisdiction: string; rate: number }[])` — Owner only. Updates the tax rate configuration.
    3. `getSalesTaxReport(startDate: string, endDate: string)` — Owner only. Summarizes sales tax collected (from invoice tax amounts via journal_entry_lines on account 2100 - Sales Tax Payable) by period. Shows: total tax collected, total tax owed, net (collected - remitted).

    **Period Close:**
    4. `getAccountingPeriods()` — Owner only. Returns all accounting_periods, ordered by period_start DESC. Shows status (open/closed), date range, closed_at, closed_by name.
    5. `createAccountingPeriod(periodStart: string, periodEnd: string)` — Owner only. Creates a new period. Validates no overlap with existing periods.
    6. `closePeriod(periodId: string)` — Owner only. Sets status='closed', closed_at=now(), closed_by=current user. After closing, the `createJournalEntry` function (from Plan 06) must check if the entry_date falls within a closed period and reject if so.
    7. `reopenPeriod(periodId: string)` — Owner only. Sets status='open', clears closed_at/closed_by. Safety net for corrections.

    **Audit Trail:**
    8. `getAuditTrail(filters: { entityType?, entityId?, userId?, startDate?, endDate? })` — Owner only. Returns a chronological log of financial actions. Since journal entries are immutable (created, never updated), the audit trail IS the journal_entries table itself — sorted by created_at, showing: date, description, source_type, source_id, created_by name, amount, is_reversed. Reversals link to the original entry.
    9. For non-journal events (period close, tax rate changes, account creation), add a lightweight `accounting_events` logging mechanism — either a dedicated table or entries in the existing alerts table with a specific type. Store: event_type, description, user_id, timestamp, metadata JSONB.

    Update the existing `createJournalEntry` in `src/lib/accounting/journal.ts` to check for closed periods:
    - Before inserting, query accounting_periods for the entry_date. If a closed period contains the entry_date, throw an error: "Cannot post to closed period {periodStart} - {periodEnd}".
