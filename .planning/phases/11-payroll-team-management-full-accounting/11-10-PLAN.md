    **New file: `src/actions/mileage.ts`**:

    1. **`calculateRouteMileage(timeEntryId: string)`** — Auto-calculates mileage for a completed shift. Gets all time_entry_stops for the shift, ordered by arrival. For each consecutive pair, calculates straight-line distance between stop coordinates using haversine (from src/lib/geo/geofence.ts), multiplies by 1.2 road distance factor. Sums total miles. Creates a mileage_logs entry with is_auto_calculated=true. Per research: "Calculate mileage at clock-out from the known stop sequence coordinates, not raw GPS breadcrumbs."

    2. **`addManualMileage(input: { workDate, originAddress, destinationAddress, purpose, miles })`** — Tech or owner creates a manual mileage entry. For non-route trips. is_auto_calculated=false.

    3. **`getMileageLog(techId?: string, startDate?: string, endDate?: string)`** — Returns mileage_logs filtered by tech and date range. Owner sees all, tech sees own.

    4. **`exportMileageLog(techId: string, year: string)`** — Generates IRS-compliant mileage log CSV: Date, Origin, Destination, Business Purpose, Miles, Rate Per Mile ($0.725 for 2026), Deduction Amount. Returns CSV string.

    5. **`getMileageSummary(startDate: string, endDate: string)`** — Total miles and deduction amount for the period. Used by financial dashboard.

    Auto-mileage calculation should be triggered on clock-out (add a call in the clockOut action from Plan 02 — add a TODO comment referencing this or wire it directly).
