    **New file: `src/lib/db/schema/time-entries.ts`** — Create 3 tables:
    1. `time_entries` — shift-level clock-in/out with GPS, status ('active'|'on_break'|'complete'), total_minutes, break_minutes, work_date (text, YYYY-MM-DD), qbo_time_activity_id, qbo_synced_at. FKs: org_id -> orgs, tech_id -> profiles. RLS: tech can read/write own entries, owner/office can read/write all in org.
    2. `break_events` — linked to time_entries. started_at, ended_at, is_auto_detected boolean. RLS: same as time_entries (tech own, owner/office all).
    3. `time_entry_stops` — per-stop timing. FKs: time_entry_id -> time_entries, route_stop_id -> route_stops. arrived_at, departed_at, onsite_minutes, drive_minutes_to_stop. RLS: same pattern.

    **New file: `src/lib/db/schema/accounting.ts`** — Create 4 tables:
    1. `chart_of_accounts` — account_number (text), account_name (text), account_type ('asset'|'liability'|'equity'|'income'|'expense'), display_name (text, pool-friendly label), parent_id (self-referencing UUID), is_system boolean (cannot delete system accounts), is_active boolean default true. FK: org_id -> orgs. RLS: owner/office read, owner write.
    2. `journal_entries` — entry_date (date), description, source_type ('invoice'|'payment'|'expense'|'payout'|'manual'|'refund'), source_id (text), is_posted boolean default true, is_reversed boolean default false, reversal_of (uuid self-ref), created_by -> profiles. RLS: owner/office read/write. IMMUTABLE pattern: no UPDATE/DELETE policy on posted entries (enforce at app layer, not RLS — RLS allows owner write but app validates is_posted=false before update).
    3. `journal_entry_lines` — journal_entry_id FK, account_id -> chart_of_accounts, amount numeric(12,2) (positive=debit, negative=credit), description. RLS: same as journal_entries.
    4. `accounting_periods` — org_id, period_start (date), period_end (date), status ('open'|'closed'), closed_at, closed_by -> profiles. RLS: owner read/write.

    **New file: `src/lib/db/schema/bank-accounts.ts`** — Create 2 tables:
    1. `bank_accounts` — plaid_item_id, plaid_access_token (NEVER return to client), plaid_cursor (for /transactions/sync), account_name, account_type ('checking'|'savings'|'credit'|'loan'), mask (last 4 digits), institution_name, current_balance numeric(12,2), available_balance numeric(12,2), chart_of_accounts_id FK, last_synced_at. RLS: owner-only for ALL operations (access token security).
    2. `bank_transactions` — bank_account_id FK, plaid_transaction_id (unique), amount numeric(12,2), date, name, merchant_name, category, pending boolean, status ('unmatched'|'matched'|'excluded'), matched_entry_id -> journal_entries, matched_at. RLS: owner/office read, owner write.

    **New file: `src/lib/db/schema/team-management.ts`** — Create 7 tables:
    1. `pto_balances` — tech_id -> profiles, pto_type ('vacation'|'sick'|'personal'), balance_hours numeric(8,2), accrual_rate_hours numeric(8,2) (per pay period), last_accrual_at. RLS: tech reads own, owner reads/writes all.
    2. `pto_requests` — tech_id -> profiles, pto_type, start_date, end_date, hours numeric(8,2), status ('pending'|'approved'|'denied'), notes, reviewed_by -> profiles, reviewed_at. RLS: tech reads/creates own, owner reads/updates all.
    3. `employee_availability` — tech_id -> profiles, day_of_week integer (0=Sun, 6=Sat), start_time text (HH:MM), end_time text (HH:MM). RLS: tech reads own, owner reads/writes all.
    4. `employee_blocked_dates` — tech_id -> profiles, blocked_date date, reason text. RLS: same pattern.
    5. `employee_documents` — tech_id -> profiles, doc_type ('cpo'|'drivers_license'|'insurance'|'other'), doc_name text, file_url text (Supabase Storage), expires_at date, notes. RLS: tech reads own, owner reads/writes all.
    6. `mileage_logs` — tech_id -> profiles, work_date text, origin_address text, destination_address text, purpose text, miles numeric(8,2), rate_per_mile numeric(6,4) default '0.7250' (2026 IRS rate), is_auto_calculated boolean, time_entry_id FK (nullable — links to shift for auto-calculated). RLS: tech reads/creates own, owner reads/writes all.
    7. `vendors` — org_id, vendor_name, contact_email, contact_phone, address, notes, is_active boolean. RLS: owner/office read/write.

    **Extend `src/lib/db/schema/org-settings.ts`** — Add Phase 11 columns after the existing Phase 10 columns:

    - `geofence_radius_meters` integer default 100

    - `overtime_threshold_hours` integer default 40
    - `accountant_mode_enabled` boolean default false
