  <action>
    **Install dependencies:** `npm install plaid react-plaid-link`

    **New file: `src/lib/plaid/client.ts`**:
    - Create and export a `plaidClient` instance using the Plaid SDK:
      ```
      import { Configuration, PlaidApi, PlaidEnvironments } from "plaid"
      const config = new Configuration({
        basePath: process.env.PLAID_ENV === "production" ? PlaidEnvironments.production : PlaidEnvironments.sandbox,
        baseOptions: { headers: { "PLAID-CLIENT-ID": process.env.PLAID_CLIENT_ID!, "PLAID-SECRET": process.env.PLAID_SECRET! } }
      })
      export const plaidClient = new PlaidApi(config)
      ```
    - Guard: if PLAID_CLIENT_ID is not set, export a null client. Functions using it should check and return error early.

    **New file: `src/actions/bank-feeds.ts`**:

    1. **`createPlaidLinkToken()`** — Owner only. Calls `plaidClient.linkTokenCreate()` with: user.client_user_id = user's profile ID, client_name = "DeweyIQ", products = ["transactions"], country_codes = ["US"], language = "en", webhook URL = `{NEXT_PUBLIC_APP_URL}/api/webhooks/plaid`. Returns { linkToken, error }.

    2. **`exchangePublicToken(publicToken: string, metadata: { accountId, accountName, accountType, mask, institutionName })`** — Owner only. Calls `plaidClient.itemPublicTokenExchange()` to get access_token + item_id. Stores in bank_accounts table via adminDb (access_token is sensitive — never withRls return). Also links to chart_of_accounts: auto-create/find a matching asset account (e.g., if account_type='checking', link to 1000 Checking). Returns { success, bankAccountId, error }.

    3. **`syncTransactions(bankAccountId: string)`** — System function (adminDb). Calls `plaidClient.transactionsSync()` with the stored cursor. Processes added/modified/removed transactions. Upserts into bank_transactions table (keyed on plaid_transaction_id). Updates the cursor. Handles pagination (loop until has_more = false). Returns count of added/modified/removed.

    4. **`getBankAccounts()`** — Owner only. Returns all bank_accounts for the org (WITHOUT plaid_access_token — never return to client). Includes: account_name, institution_name, mask, type, current_balance, last_synced_at.

    5. **`disconnectBankAccount(bankAccountId: string)`** — Owner only. Calls `plaidClient.itemRemove()` to revoke access. Deletes bank_account row and associated bank_transactions. Returns { success, error }.

    6. **`refreshBankBalance(bankAccountId: string)`** — System function. Calls `plaidClient.accountsBalanceGet()` and updates current_balance, available_balance on bank_accounts row.

    CRITICAL: plaid_access_token is NEVER returned from any server action. It is only read server-side for Plaid API calls.
