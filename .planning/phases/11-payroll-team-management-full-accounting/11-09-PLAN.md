  <name>Task 1: Build reconciliation matching algorithm and server actions</name>
  <files>

  <action>
    **New file: `src/lib/accounting/reconciliation.ts`** — Pure matching logic:

    1. **`scoreBankTransactionMatch(bankTxn, journalEntry)`** — Returns a numeric score (0-100) based on:
       - Amount exact match (±$0.01): +50 points
       - Date within 1 day: +30 points; within 3 days: +20 points; within 7 days: +10 points
       - Description contains invoice number: +40 points
       - Description contains customer/vendor name: +20 points
       Score thresholds: >= 80 = auto-match; 50-79 = suggest for review; < 50 = unmatched.

    2. **`matchBankTransaction(bankTxnId: string, orgId: string)`** — Fetches the bank transaction, finds unmatched journal entries for the org, scores each, returns top matches with scores. Does NOT auto-match (caller decides).

    3. **`autoMatchTransactions(bankAccountId: string)`** — Batch process: for all unmatched bank_transactions on this account, run scoring against unmatched journal entries. Auto-match those scoring >= 80. Return counts: { autoMatched, suggestedReview, unmatched }.

    **New file: `src/actions/reconciliation.ts`**:

    1. **`getReconciliationView(bankAccountId: string, filters?: { status?, dateRange? })`** — Owner only. Returns bank transactions for an account with their match status. Includes: matched entries (with journal entry details), suggested matches (score 50-79), and unmatched transactions. Paginated.

    2. **`confirmMatch(bankTxnId: string, journalEntryId: string)`** — Owner only. Sets bank_transaction.matched_entry_id, status='matched', matched_at. Validates: neither the bank txn nor the journal entry is already matched elsewhere.

    3. **`unmatchTransaction(bankTxnId: string)`** — Owner only. Clears matched_entry_id, sets status='unmatched'.

    4. **`excludeTransaction(bankTxnId: string)`** — Owner only. Sets status='excluded'. For transactions that don't belong in reconciliation (transfers between own accounts, personal charges on business card, etc.).

    5. **`createEntryFromTransaction(bankTxnId: string, accountId: string, description: string)`** — Owner only. Creates a journal entry from an unmatched bank transaction (e.g., a new expense). Dr: selected expense account, Cr: bank account. Then auto-matches the new entry to the bank transaction.

    6. **`runAutoMatch(bankAccountId: string)`** — Owner only. Triggers autoMatchTransactions and returns results.
  </action>

    2. Score function produces correct values for test cases
    3. Auto-match only matches at score >= 80
  </verify>
