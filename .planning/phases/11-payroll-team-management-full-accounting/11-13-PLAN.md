<task type="auto">
  <name>Task 1: Create team dashboard and labor cost server actions</name>
  <files>src/actions/team-dashboard.ts</files>
  <action>
    Create `src/actions/team-dashboard.ts`:

    1. **`getTeamDashboard()`** — Owner only. Returns per-employee data:
       - Profile info: name, email, role, avatar
       - Current status: derived from time_entries — 'off_shift' (no active entry), 'clocked_in' (active entry, status='active'), 'on_break' (status='on_break'), 'clocked_out' (most recent entry completed today)
       - Today's hours: sum of completed time_entries for today + current shift elapsed time
       - This week's hours: sum of all time_entries for current week (Mon-Sun)
       - PTO balance: from pto_balances (vacation + sick total)
       - Pending PTO requests count
       - Expiring certifications: count of employee_documents expiring within 30 days
       - Stops completed today: count of completed service_visits by this tech today
       Uses withRls + LEFT JOINs. Single query if possible, otherwise 2-3 queries with batched profile IDs.

    2. **`getLaborCostAnalysis(startDate: string, endDate: string)`** — Owner only. Returns labor cost breakdown:
       - **Per stop**: For each stop in the date range, calculate: onsite_minutes (from time_entry_stops), tech pay_rate, estimated cost = (onsite_minutes/60) * pay_rate. Group by customer for per-customer view.
       - **Per route**: For each tech+date combo, total labor hours * pay_rate.
       - **Per customer**: Aggregate all stop costs by customer for the date range.
       - **Summary**: Total labor cost, average cost per stop, most expensive routes/customers.
       Note: pay_rate is from profiles table (Phase 9). Per CONTEXT, detailed pay rates live in QBO, but the local pay_rate is used for estimation/reporting (orgs without QBO use this for payroll CSV export).

    3. **`getTeamAlerts()`** — Owner only. Returns: certification expiry alerts, break compliance violations, open PTO requests, unresolved clock-out-forgotten entries (clocked_in_at set, clocked_out_at NULL, work_date < today).
  </action>

  </verify>
