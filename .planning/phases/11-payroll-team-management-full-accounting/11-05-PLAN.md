<task type="auto">
  <name>Task 1: Create team management server actions</name>
  <files>src/actions/team-management.ts</files>
  <action>
    Create `src/actions/team-management.ts` with server actions:

    **PTO:**
    1. `getPtoBalances(techId?: string)` — Owner sees all tech PTO balances; tech sees only own. Returns pto_balances rows joined with profile names.
    2. `updatePtoBalance(techId: string, ptoType: string, balanceHours: number, accrualRateHours: number)` — Owner only. Upserts pto_balances row.
    3. `requestPto(input: { ptoType, startDate, endDate, hours, notes })` — Tech creates a pto_request with status='pending'. Creates alert for owner: "PTO Request from {techName}".
    4. `approvePto(requestId: string, approved: boolean)` — Owner only. Sets status to 'approved' or 'denied', records reviewed_by and reviewed_at. If approved, deducts hours from pto_balances. Creates notification for the tech.
    5. `getPtoRequests(status?: string)` — Owner gets all, tech gets own. Filterable by status.

    **Availability:**
    6. `getAvailability(techId: string)` — Returns employee_availability rows (day_of_week + time windows) and employee_blocked_dates.
    7. `updateAvailability(techId: string, windows: { dayOfWeek: number; startTime: string; endTime: string }[])` — Owner only. Replaces all availability windows for a tech (delete + insert pattern).
    8. `addBlockedDate(techId: string, date: string, reason: string)` — Owner only. Adds a blocked date.
    9. `removeBlockedDate(id: string)` — Owner only. Removes a blocked date.

    **Certifications/Documents:**
    10. `getDocuments(techId?: string)` — Owner sees all, tech sees own. Joins with profiles for names.
    11. `uploadDocument(input: { techId, docType, docName, expiresAt?, notes? })` — Owner only. Creates employee_documents row. Returns a signed upload URL for the file (Supabase Storage, bucket: 'employee-documents', path: org_id/tech_id/filename).
    12. `deleteDocument(docId: string)` — Owner only. Soft delete or hard delete + remove from Storage.
    13. `checkExpiringDocuments()` — System function (adminDb). Finds all documents expiring within 30 days. Creates alerts for the owner. Can be called from a cron edge function.

    All actions: getRlsToken() + role checks. Tech actions restricted to own data. Owner can manage all team members.
  </action>
  <verify>
    `npm run build` passes — all actions compile with correct types
  </verify>
  <done>
    13 server actions covering PTO balance management, request/approval workflow, availability windows, blocked dates, and document upload/tracking with expiry alerts.
  </done>
