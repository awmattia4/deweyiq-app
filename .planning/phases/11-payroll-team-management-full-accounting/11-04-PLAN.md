  <action>
    **New file: `src/actions/timesheets.ts`**:

    1. **`getTimesheets(weekStartDate: string, techId?: string)`** — Owner/office only. Returns time_entries for a given week (Mon-Sun) grouped by tech. Each tech entry includes: daily hours breakdown (Mon-Sun), total regular hours, overtime hours (calculated against org_settings.overtime_threshold_hours, default 40), total break minutes, per-stop timing breakdown (from time_entry_stops), PTO hours for the week (from pto_requests with status='approved'). If techId provided, filter to that tech. Uses withRls + LEFT JOIN on break_events and time_entry_stops.

    2. **`editTimeEntry(timeEntryId: string, updates: { clocked_in_at?: Date; clocked_out_at?: Date })`** — Owner only. Allows manual correction of clock times. Recalculates total_minutes and break_minutes. Only allowed if time_entry has NOT been synced to QBO yet (qbo_time_activity_id IS NULL). Returns { success, error }.

    3. **`approveTimesheet(techId: string, weekStartDate: string)`** — Owner only. Marks all time_entries for the tech in that week as approved (add an `approved_at` timestamp and `approved_by` to time_entries if not already there — or track via a flag). After approval, triggers QBO push for each entry.

    4. **`getTimesheetSummary(startDate: string, endDate: string)`** — Owner only. Returns aggregated timesheet data for a date range (pay period). Used for the payroll export view. Extends/replaces Phase 9's `getPayrollPrep()` — when time tracking is enabled, uses real time_entry data; when disabled, falls back to the old route_stops-based calculation.

    **New file: `src/lib/qbo/time-sync.ts`**:

    1. **`syncEmployeeToQbo(profileId: string)`** — Creates or updates a QBO Employee entity for a tech profile. Uses adminDb to read profile, calls `qbo.createEmployee()` or `qbo.updateEmployee()` via the existing `qboPromise()` pattern. Stores the returned QBO Employee ID in `profiles.qbo_employee_id`. If QBO is not connected for the org, no-op. Fire-and-forget pattern.

    2. **`pushTimeEntryToQbo(timeEntryId: string)`** — Pushes a single time_entry as a QBO TimeActivity. Requires: tech has qbo_employee_id (if not, calls syncEmployeeToQbo first). Maps: work_date -> TxnDate, total_minutes -> Hours+Minutes, EmployeeRef from qbo_employee_id. Uses `qbo.createTimeActivity(payload, cb)`. Stores qbo_time_activity_id and qbo_synced_at on success. Fire-and-forget — never throw. Log errors.

    3. **`pushPayPeriodToQbo(techId: string, weekStartDate: string)`** — Batch push: gets all approved, un-synced time_entries for the tech in the given week. Calls pushTimeEntryToQbo for each. Returns count of pushed/failed.

    **Extend `src/lib/qbo/mappers.ts`** — Add:
    - `mapTimeEntryToQboTimeActivity(entry, qboEmployeeRef)` — Maps a time_entry row to a QBO TimeActivity payload object. Fields: TxnDate (work_date), NameOf: "Employee", EmployeeRef: { value: qboEmployeeRef }, Hours, Minutes, Description: `Field route - {work_date}`, BillableStatus: "NotBillable".
    - `mapProfileToQboEmployee(profile)` — Maps a profiles row to a QBO Employee create payload. Fields: DisplayName, PrimaryEmailAddr, GivenName, FamilyName.

    Per CONTEXT OVERRIDE for TEAM-07: Commission structures live in QBO. Phase 9 already tracks upsell events via `wo_upsell_commission_pct`. No new commission logic needed — the existing event data is sufficient for QBO.
