  <action>
    **Extend `src/lib/db/schema/payments.ts`** — Add new tables (or create in a new file if payments.ts is already complex):
    1. `payment_plans` — invoice_id FK, total_amount, installment_count, installment_amount, frequency ('weekly'|'bi_weekly'|'monthly'), start_date, status ('active'|'completed'|'cancelled'), created_by -> profiles. RLS: owner/office read/write.
    2. `payment_plan_installments` — payment_plan_id FK, installment_number, due_date, amount, status ('pending'|'paid'|'overdue'), payment_record_id FK (nullable, linked when paid). RLS: owner/office read/write.
    3. `customer_credits` — customer_id FK, org_id, amount numeric(12,2), reason text, source_type ('refund'|'goodwill'|'overpayment'), source_id (nullable), applied_to_invoice_id FK (nullable), status ('available'|'applied'|'expired'), created_by, created_at. RLS: owner/office read/write.

    **New file: `src/actions/payment-reconciliation.ts`**:

    **QBO Payment Reconciliation:**
    1. `onQboPaymentReceived(paymentData: QboPaymentWebhook)` — Triggered by existing QBO webhook handler. Creates journal entry: Dr Bank/Clearing, Cr AR. Links to the invoice if identifiable.

    **Payment Plans:**
    2. `createPaymentPlan(invoiceId: string, installments: number, frequency: string)` — Owner only. Splits invoice total into equal installments. Creates payment_plan + payment_plan_installments rows. Returns plan with schedule.
    3. `getPaymentPlans(customerId?: string)` — Owner/office. Returns active payment plans with installment status.
    4. `recordInstallmentPayment(installmentId: string, paymentRecordId: string)` — Marks installment as paid. If all installments paid, complete the plan.

    **Customer Credits:**
    5. `issueCustomerCredit(customerId: string, amount: string, reason: string, sourceType: string)` — Owner only. Creates customer_credits row with status='available'. Generates journal entry: Dr Customer Credits (liability 2200), Cr appropriate source account.
    6. `applyCustomerCredit(creditId: string, invoiceId: string)` — Owner only. Applies credit as offset on invoice. Updates credit status to 'applied', sets applied_to_invoice_id. Adjusts invoice remaining balance. Generates journal entry: Dr AR (reduce), Cr Customer Credits (reduce liability).
    7. `getCustomerCredits(customerId: string)` — Returns all credits for a customer with status.

    **Collections Dashboard:**
    8. `getCollectionsDashboard()` — Owner only. Returns: customers with invoices overdue > 30/60/90 days, failed autopay attempts, customers with multiple missed payments, total outstanding AR. Groups by severity bucket.
    9. `getArApAging()` — Owner only. Returns AR aging buckets (current, 30, 60, 90, 120+ days) with customer breakdown. Also AP aging if vendor bills exist. Extends Phase 7's `getArAging()`.

    **Refund Journal Entries:**
    10. `createRefundEntry(paymentId: string, refundAmount: string)` — Creates reversal journal entry for refund. Dr Revenue (or appropriate income account), Cr Bank/Stripe. Integrates with existing Phase 7 refund handling.

    PAY-01 (Stripe Connect): Already implemented in Phase 7 — acknowledge no new work needed.
