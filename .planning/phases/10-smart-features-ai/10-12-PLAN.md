    **ETA calculator** — Create `src/lib/eta/calculator.ts`:
    - `computeEta(techPosition: { lat, lng }, remainingStops: EtaStop[], avgServiceMinutes?: number)`:
      - For each remaining stop, sum: drive time from previous position to stop (via ORS getRouteDirections or haversine fallback for offline), plus service duration (per-pool historical avg or org default 25 min).
      - Returns `Map<stopId, { etaMinutes: number; etaTime: Date }>`.
      - Accept optional `avgServiceMinutes` override (from org_settings or per-pool computed).

    **ETA actions** — Create `src/actions/eta.ts`:
    - `computeRouteEtas(techId: string, date: string)`: Queries tech's route stops for the day, gets tech's latest GPS from dispatch Realtime channel (or DB cache), computes ETA for each uncompleted stop. Returns array of `{ stopId, customerId, etaMinutes, etaTime }`.
    - `sendEtaNotification(stopId: string, etaMinutes: number, notificationType: 'initial' | 'refined' | 'update')`:
      - Checks `eta_sms_count` on the route_stop (add this column if not exists — smallint default 0).
      - If notificationType is 'update' and eta_sms_count >= 2, skip (capped at 2 updates per user decision).
      - If notificationType is 'update' and shift < 15 minutes from previous ETA, skip (threshold per user decision).
      - Resolve ETA notification template (`eta_initial_sms`, `eta_update_sms`) with merge tags: `customer_name`, `company_name`, `tech_name`, `eta_time`, `eta_minutes`.
      - Send SMS/email via Edge Function pattern.
      - Increment eta_sms_count on the route_stop.
    - `triggerEtaNotifications(techId: string)`: Called from startRoute and on stop completion. Computes ETAs for remaining stops. For each customer:
      - At route start: send 'initial' ETA to all customers.
      - When tech is 2-3 stops away: send 'refined' ETA.
      - When ETA shifts 15+ minutes: send 'update' (if under cap).

    **ETA templates** — Add to default-templates.ts:
    - `eta_initial_sms`: "{{company_name}}: Your pool tech {{tech_name}} has started their route. Estimated arrival: {{eta_time}}."
    - `eta_update_sms`: "{{company_name}}: Updated arrival time: {{eta_time}} (was {{previous_eta_time}}). {{tech_name}} is on the way."

    **Dispatch overlay** — Create `src/components/dispatch/eta-overlay.tsx`:

    - Fetches ETAs from `computeRouteEtas` and displays: stop name, customer name, ETA time, ETA minutes from now.
    - Auto-refreshes every 60 seconds (or on GPS update if available via Realtime).

    **Dispatch integration** — Update `src/app/(app)/dispatch/dispatch-client-shell.tsx`:
    - When a tech is selected and has an active route, show the EtaOverlay panel.
