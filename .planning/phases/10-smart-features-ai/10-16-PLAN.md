  <files>src/actions/broadcast.ts</files>
  <action>
    Create `src/actions/broadcast.ts`:

    1. `getSegmentCount(segment: BroadcastSegment)`: Returns the count of customers matching the segment. Used for preview before send.
       - Segments:
         - `{ type: 'all' }` — all customers with notifications_enabled = true
         - `{ type: 'active' }` — customers with at least one route_stop in the last 30 days
         - `{ type: 'tech_route', techId: string }` — customers assigned to a specific tech's routes
         - `{ type: 'individual', customerIds: string[] }` — specific customers
       - Uses adminDb. Returns `{ count, hasEmail, hasPhone }` (how many have email, how many have phone).

    2. `sendBroadcast(params: { segment, subject: string, bodyHtml: string, smsText: string, channels: ('email' | 'sms')[] })`:
       - Owner-only (check JWT role).
       - Query customers matching segment.
       - For each customer (in batches of 50 to avoid timeouts):
         - If 'email' in channels and customer has email: send via Resend (invoke Edge Function or direct API).
         - If 'sms' in channels and customer has phone: send via Twilio Edge Function.
       - Use `Promise.allSettled` per batch.
       - Track results: `{ totalTargeted, emailSent, emailFailed, smsSent, smsFailed }`.
       - Store broadcast record in a simple JSONB field on org_settings (or a dedicated broadcasts table — use JSONB for simplicity since we don't need complex queries on broadcast history).
       - Return results summary.

    3. `getBroadcastHistory()`: Returns recent broadcasts (last 10) with send counts and dates. Owner-only.

    Note: Merge tags in the broadcast body (`{{customer_name}}`, `{{company_name}}`) are resolved per customer using the existing `resolveTemplate` function from template-engine.ts.
  </action>
