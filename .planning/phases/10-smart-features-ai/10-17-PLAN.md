<task type="auto">
  <name>Task 1: Service worker push handler and client subscription</name>
  <files>src/app/sw.ts, src/lib/push/subscribe.ts</files>
  <action>
    **Extend service worker** — Update `src/app/sw.ts`:
    - After the existing Serwist initialization and `serwist.addEventListeners()`, add:
    - `self.addEventListener('push', ...)`:
      - Parse `event.data.json()` for `{ title, body, url, icon }`.
      - Call `self.registration.showNotification(data.title, { body: data.body, icon: data.icon || '/icons/icon-192x192.png', badge: '/icons/badge-72x72.png', data: { url: data.url }, vibrate: [100, 50, 100] })`.
      - Wrap in `event.waitUntil()`.
    - `self.addEventListener('notificationclick', ...)`:
      - Close notification.
      - Open or focus the app window with the notification's URL.
      - Use `clients.matchAll()` to find existing window and focus it, or `clients.openWindow(url)` if none.
      - Wrap in `event.waitUntil()`.

    **Client subscription helper** — Create `src/lib/push/subscribe.ts`:
    - `urlBase64ToUint8Array(base64String)`: Converts VAPID public key from base64url to Uint8Array for `pushManager.subscribe()`.
    - `subscribeToPush()`:
      1. Check `'serviceWorker' in navigator && 'PushManager' in window`. Return early if not supported.
      2. Get SW registration via `navigator.serviceWorker.ready`.
      3. Check existing subscription via `registration.pushManager.getSubscription()`.
      4. If no subscription, call `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!) })`.
      5. Serialize subscription and call `subscribeUserPush()` server action (from Plan 10-09) to persist.
    - `unsubscribeFromPush()`:
      1. Get existing subscription.
      2. Call `subscription.unsubscribe()`.
      3. Call `unsubscribeUserPush(subscription.endpoint)` server action.
    - `isPushSubscribed()`: Returns boolean — checks if subscription exists.

    Note per research Pitfall 1: On iOS, push only works in installed PWA mode. Check `window.matchMedia('(display-mode: standalone)').matches` before attempting subscription on iOS. If not installed, skip push subscription silently (the install prompt will guide them).
  </action>
  <verify>Run `npm run build` — compiles. Check that sw.ts builds into the service worker bundle.</verify>
  <done>Service worker handles push events and shows native notifications. Client subscription helper manages VAPID subscription lifecycle.</done>
</task>
