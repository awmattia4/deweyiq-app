  <files>src/lib/db/schema/equipment-readings.ts, src/lib/db/schema/index.ts, src/lib/db/schema/relations.ts, src/actions/equipment-readings.ts</files>
  <action>
    **Schema** — Create `src/lib/db/schema/equipment-readings.ts`:
    ```
    equipment_readings table:
    - id: uuid PK
    - org_id: uuid FK(orgs) not null
    - equipment_id: uuid FK(equipment) not null
    - pool_id: uuid FK(pools) not null
    - service_visit_id: uuid FK(service_visits) (optional, links to the visit when reading was taken)
    - recorded_at: timestamp default now()
    - metrics: jsonb not null (equipment-type-specific: { salt_ppm, filter_psi, pump_flow_gpm, pump_rpm, heater_delta_f, etc. })
    - recorded_by_id: uuid FK(profiles)
    - notes: text
    - created_at: timestamp default now()
    ```
    RLS: SELECT/INSERT/UPDATE for all org members (techs log, office views). Export and add to schema index.ts. Add relations in relations.ts.

    **Server actions** — Create `src/actions/equipment-readings.ts`:
    - `logEquipmentReading(equipmentId, poolId, metrics, visitId?, notes?)`: INSERT reading. Uses withRls.
    - `getEquipmentReadings(equipmentId, limit?)`: Returns readings for an equipment piece, ordered by recorded_at desc. Uses withRls.
    - `getEquipmentHealth(equipmentId)`: Computes health score:
      1. Get last 8 readings for the equipment.
      2. For each metric key in the JSONB, compute: baseline (average of first 4 readings), current (average of last 2 readings).
      3. If current < baseline * 0.70 (30% drop), mark as "degraded". If current < baseline * 0.50, mark as "critical".
      4. Return `{ metricName, baseline, current, status: 'healthy' | 'degraded' | 'critical', dropPct }[]`.
    - `checkDegradation(orgId)`: Run across all equipment in the org. For each equipment piece with 8+ readings, check for degradation. Create alerts for degraded equipment. Alert includes equipment name, pool name, customer name, metric that degraded, and percentage drop. Alert metadata includes `{ equipmentId, createWoLink: '/work-orders?create=true&equipment=${equipmentId}' }` for the one-tap WO creation button.

    Run `drizzle-kit generate` for migration. Verify RLS policies.
  </action>
  <verify>Run `npx drizzle-kit generate` — migration generated. Run `npx tsc --noEmit` — no type errors.</verify>
  <done>Equipment readings table exists. Server actions support logging, health scoring, and degradation detection.</done>
