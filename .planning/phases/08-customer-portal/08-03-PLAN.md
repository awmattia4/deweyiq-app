**Add to `src/actions/portal-data.ts`:**

`getCustomerInvoices(orgId: string, customerId: string)`:
- Uses `adminDb`. Query: SELECT invoices.*, invoice_line_items FROM invoices LEFT JOIN invoice_line_items ON invoice_line_items.invoice_id = invoices.id WHERE invoices.customer_id = customerId AND invoices.org_id = orgId AND invoices.status IN ('sent', 'paid') ORDER BY invoices.created_at DESC.
- Do NOT show 'draft' or 'void' invoices to customers — only 'sent' and 'paid'.
- Group line items per invoice in the return value.
- Also query payment_records for each invoice to show payment history.
- Return type: `PortalInvoice[]` with nested `lineItems` and `payments`.

`createPortalPaymentIntent(orgId: string, customerId: string, invoiceId: string)`:
- Validates: invoice belongs to customerId, invoice is in 'sent' status, not already paid.
- Gets `stripe_account_id` from org_settings, `stripe_customer_id` from customers.
- If no `stripe_customer_id`, create one: `stripe.customers.create({ email, name }, { stripeAccount })` and save to customers table.
- Creates PaymentIntent: `stripe.paymentIntents.create({ amount (in cents), currency: 'usd', customer: stripeCustomerId, metadata: { invoice_id, org_id } }, { stripeAccount })`.
- **CRITICAL (from research Pitfall 6):** Always pass `{ stripeAccount: org_settings.stripe_account_id }` — the PI must be on the connected account.
- Include surcharge if `cc_surcharge_enabled` (from org_settings) — match the pattern from `/pay/[token]` page.
- Returns `{ clientSecret, publishableKey, stripeAccount, amount, surchargeAmount }`.

`createPortalSetupIntent(orgId: string, customerId: string)`:
- Gets `stripe_account_id` from org_settings and `stripe_customer_id` from customers.
- If no stripe_customer_id, create one (same as above).
- `stripe.setupIntents.create({ customer: stripeCustomerId, usage: 'off_session', automatic_payment_methods: { enabled: true } }, { stripeAccount })`.
- Returns `{ clientSecret, publishableKey, stripeAccount }`.
- After successful setup, the webhook will save the payment method to the customer. But also provide a `confirmPaymentMethodUpdate(customerId, paymentMethodId)` action that updates `customers.autopay_method_id`.

`updateCustomerContactInfo(orgId: string, customerId: string, data: { phone?: string, email?: string })`:
- Uses `adminDb` to update customers table with provided fields.
- Only allows updating `phone` and `email` — customers cannot change their name, address, or billing fields.
- Returns `{ success: true }`.

`getCustomerPaymentMethods(orgId: string, customerId: string)`:
- Gets stripe_customer_id and stripe_account_id.
- Lists payment methods: `stripe.paymentMethods.list({ customer: stripeCustomerId, type: 'card' }, { stripeAccount })`.
- Also list 'us_bank_account' type for ACH.
- Returns formatted list with: last4, brand, exp_month, exp_year, type, id.
- Indicates which is the default/autopay method by comparing to `customers.autopay_method_id`.

**Create `src/components/portal/payment-form.tsx`:**
