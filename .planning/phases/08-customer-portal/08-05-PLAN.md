**Create `src/actions/portal-messages.ts`:**

`sendMessage(params: { orgId: string, customerId: string, senderRole: 'customer' | 'office', senderName: string, body: string | null, photoPath: string | null })`:
- Uses `adminDb` to INSERT into portal_messages.
- After insert, broadcast the message via Supabase Realtime to the channel `portal-thread-${customerId}`:
  ```typescript
  const supabase = createAdminClient()  // or use the service role client
  await supabase.channel(`portal-thread-${customerId}`).send({
    type: 'broadcast',
    event: 'message',
    payload: { id: newMessage.id, body, senderRole, senderName, photoPath, created_at: new Date().toISOString() },
  })
  ```
- Send email notification to the OTHER party:
  - If `senderRole === 'customer'`: send email to office (get office email from org_settings or the first owner/office profile).
  - If `senderRole === 'office'`: send email to customer (get customer email from customers table).
  - Email content: "{senderName} sent you a message: {body preview}. View in portal: {link}."
  - Use Resend for email delivery (same pattern as existing notification emails).
  - Use the notification template system if a template exists for 'portal_message'.
- Returns the created message object.

`getMessages(orgId: string, customerId: string, limit?: number)`:
- Uses `adminDb`. SELECT * FROM portal_messages WHERE customer_id = customerId AND org_id = orgId ORDER BY created_at ASC LIMIT (limit || 100).
- For any messages with `photo_path`, generate signed URLs (1-hour expiry).
- Returns `PortalMessage[]`.

`getInboxThreads(orgId: string)`:
- Uses `withRls` (office/owner only).
- Query: For each unique customer_id in portal_messages, get: customer name (join customers), last message body, last message time, unread count (WHERE read_by_office_at IS NULL AND sender_role = 'customer').
- SQL approach: GROUP BY customer_id with aggregate functions.
- Sort by last message time DESC (newest threads first), with unread threads at top.
- Returns `InboxThread[]`.

`markAsRead(orgId: string, customerId: string, role: 'office' | 'customer')`:
- Uses `adminDb`. UPDATE portal_messages SET read_by_office_at = NOW() (if role='office') or read_by_customer_at = NOW() (if role='customer') WHERE customer_id = customerId AND org_id = orgId AND the corresponding read_at is NULL AND sender_role != role (only mark the OTHER party's messages as read).
- Called when user opens a thread.

`getUnreadCount(orgId: string, role: 'office' | 'customer', customerId?: string)`:
- For office: count all messages WHERE sender_role = 'customer' AND read_by_office_at IS NULL AND org_id = orgId.
- For customer: count all messages WHERE sender_role = 'office' AND read_by_customer_at IS NULL AND customer_id = customerId AND org_id = orgId.
- Returns `{ count: number }`.

`createMessagePhotoUploadUrl(orgId: string, customerId: string, filename: string)`:
- Generates signed upload URL: `supabase.storage.from('company-assets').createSignedUploadUrl(`${orgId}/portal/messages/${Date.now()}-${filename}`)`.
- Returns `{ signedUrl, path }`.

**Type definitions:**
```typescript
export interface PortalMessage {
  id: string
  sender_role: 'customer' | 'office'
  sender_name: string
  body: string | null
  photo_url: string | null  // signed URL (resolved from photo_path)
  photo_path: string | null
  created_at: string
}

export interface InboxThread {
  customerId: string
  customerName: string
  customerEmail: string
  lastMessage: string | null
  lastMessageAt: string
  unreadCount: number
}
```
