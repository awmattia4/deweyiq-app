  <action>
    Create `src/actions/billing.ts` with "use server" directive:

    1. `getSubscription()` — fetches the current org's subscription from local DB via withRls
       - Returns subscription row or null
       - Used by billing page and layout for banner display

    2. `createTrialSubscription(orgId: string, orgName: string, ownerEmail: string)` — called on org creation
       - Creates a Stripe Customer with org name and email
       - Inserts subscription row with status "trialing", trial_ends_at = now + 14 days, tier "starter", pool_limit 79
       - Uses adminDb (no RLS — called during signup before user has JWT claims)

    3. `createCheckoutSession(tier: BillingTier, interval: BillingInterval)` — creates Stripe Checkout Session
       - Validates user is owner role
       - Fetches subscription to get stripe_customer_id
       - Creates Stripe Checkout Session with:
        - mode: "subscription"
        - customer: stripe_customer_id
        - line_items: [{ price: TIER_PRICE_IDS[tier][interval], quantity: 1 }]
        - success_url: `{APP_URL}/billing/checkout/success?session_id={CHECKOUT_SESSION_ID}`
        - cancel_url: `{APP_URL}/billing/checkout/cancel`
        - If currently trialing: set subscription_data.trial_end to preserve remaining trial
       - Returns { url: session.url } for client-side redirect

    4. `getSubscriptionStatus()` — lightweight check for layout banners
       - Returns { status, tier, trialEndsAt, cancelAtPeriodEnd, poolCount, poolLimit, failedPaymentCount, gracePeriodEndsAt } or null
       - Used by app layout to determine which banners to show

    Import helpers from src/lib/stripe.ts and schema from src/lib/db/schema.
  </action>
  <verify>

    2. All four functions exported correctly

    4. createTrialSubscription uses adminDb
  </verify>
