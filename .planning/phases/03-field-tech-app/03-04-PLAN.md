**1. Create ChemistryGrid component** in `src/components/field/chemistry-grid.tsx`:
- Per locked decision: "Quick-entry grid: all chemistry parameters visible in a compact grid, tap any cell to enter a value"
- Grid layout: 2 columns on mobile (parameter name | current value), with a muted "last" column showing previous visit's reading
- Per locked decision: "Previous visit's readings shown side by side in a muted column next to current entry fields"
- Parameters displayed (in order):
  1. Free Chlorine (FC) — ppm
  2. Combined Chlorine (CC) — ppm
  3. pH
  4. Total Alkalinity (TA) — ppm
  5. Cyanuric Acid (CYA) — ppm
  6. Calcium Hardness (CH) — ppm
  7. TDS — ppm
  8. Phosphates — ppm
  9. Salt — ppm
  10. Temperature — F
- Each cell: `<input type="text" inputMode="decimal" pattern="[0-9]*\.?[0-9]*">` (per Research pitfall 6 — inputMode="decimal" for iOS decimal keypad, NOT type="number")
- Per locked decision: "Out-of-range readings: color-coded cells (green/yellow/red) plus badge text ('LOW' / 'HIGH') next to the value"
  - Import `classifyReading` from `lib/chemistry/targets`
  - Green border + no badge = in range
  - Yellow border + "LOW"/"HIGH" badge = slightly out of range
  - Red border + "LOW"/"HIGH" badge = significantly out of range
- Per locked decision: "Which chemistry parameters are required vs optional is configurable per customer/pool by the office"
  - For now, mark required parameters with a subtle asterisk; required params vary by sanitizer type (e.g., CYA not required for bromine)
  - Default required: FC (or Bromine if bromine sanitizer), pH, TA
- On value change: write to Dexie `visitDrafts.chemistry` immediately via `updateChemistry()` from the draft hook
- No form submission — each cell change is saved independently to Dexie
- Use plain React state for input focus management (not react-hook-form — per locked pattern from Phase 2)

**2. Create ChemistryDosing component** in `src/components/field/chemistry-dosing.tsx`:
- Per locked decision: "LSI value and dosing recommendations appear inline below the chemistry grid, updating live as readings are entered"
- **LSI/CSI Section:**
  - Import `calculateCSI` and `interpretCSI` from `lib/chemistry/lsi`
  - Calculate CSI on every render (pure function — no network, instant)
  - Display CSI value with color indicator: green for balanced, yellow for slightly off, red for corrosive/scaling
  - Show text label: "Balanced", "Slightly Corrosive", "Corrosive", "Slightly Scaling", "Scaling"
  - If insufficient readings for CSI calc (returns null), show "Enter pH, TA, CH, and Temp to see LSI"
- **Dosing Recommendations Section:**
  - Import `generateDosingRecommendations` from `lib/chemistry/dosing`
  - Pass current readings, pool info (volume, sanitizer type), and org's chemical products
  - Per locked decision: "Exact dosing amounts (e.g., '12 oz muriatic acid'), not ranges"
  - Per locked decision: "Fluid ounces for liquid chemicals, pounds for dry chemicals"
  - Display each recommendation as a row: chemical product name, amount with unit, action verb ("Add" or "Use"), and reason
  - Example: "Add 12 fl oz Muriatic Acid (31.45%) — Lower pH from 7.9 to 7.5"
  - If all readings are in range: show "All readings in range" with green check
  - If no readings entered: show "Enter readings to see recommendations"
- Per locked decision: "Product-aware dosing: office configures which chemical products the company uses (including concentration percentages); doses adjust based on actual product concentration"
  - Products come from the stop context (fetched in Task 1)
  - If no products configured for the org, show a note: "No chemical products configured — ask your office to set up products"

All values computed client-side. Zero network dependency. Works completely offline.
