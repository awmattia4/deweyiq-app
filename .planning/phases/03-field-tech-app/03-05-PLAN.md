**1. Create Checklist component** in `src/components/field/checklist.tsx`:

Props: `{ tasks: ChecklistTask[], draft: VisitDraft, onUpdate: (checklist) => void }`

The `tasks` array comes from the stop context (fetched in 03-04 Task 1): merged template tasks + customer overrides, sorted by `sort_order`. Customer overrides with `is_deleted: true` are excluded.

**Layout:**
- Per locked decision: "'Mark all complete' button at the top for routine visits where everything was done"
  - Large green button at top of list: "Mark All Complete" with checkmark icon
  - Tapping it checks all unchecked tasks at once
  - If all already checked, button is disabled with "All tasks complete" text
  - 44px minimum height, full-width

- Task list below the mark-all button:
  - Per locked decision: "Each checklist task has a checkbox AND an optional notes field for exceptions"
  - Each task row:
    - Left: shadcn `Checkbox` (44px tap target)
    - Center: task label text (e.g., "Skim surface", "Brush walls", "Vacuum")
    - When task is checked: label gets subtle strikethrough + muted color
    - Below checked task (expandable): optional textarea for exception notes
      - Collapsed by default — tapping a small "Add note" text link expands it
      - Per context: "e.g., 'filter pressure high — needs replacement'"
      - Placeholder: "Add notes about this task..."
    - 44px minimum row height, generous vertical padding between tasks

- If no tasks in the checklist (empty template): show "No checklist configured for this service type. Ask your office to set up templates."

**Offline persistence:**
- Every checkbox toggle and note edit writes immediately to `draft.checklist` in Dexie via `onUpdate()`
- Checklist state format in VisitDraft: `Array<{ taskId: string, completed: boolean, notes: string }>`
- On mount, merge stored draft state with template tasks:
  - If draft has state for a taskId → use draft values (preserves work-in-progress)
  - If template has a task not in draft → initialize as unchecked with empty notes
  - If draft has a taskId not in template → keep it (handles template changes gracefully)
