**1. Create GET `/api/routes/today` endpoint** in `src/app/api/routes/today/route.ts`:
- Authenticate via Supabase server client
- Get the authenticated user's profile to determine tech_id and org_id
- Query `route_days` for today's date + tech_id using `withRls()`
- If a route_day exists, parse the `stop_order` JSONB to get customer/pool IDs in order
- For each stop in order, fetch customer (full_name, address, phone, gate_code, access_notes, status) and pool (name, type, volume_gallons, sanitizer_type, notes) data using LEFT JOIN (not correlated subquery — RLS pitfall)
- Also fetch the last service_visit for each pool (visited_at, status) for "last service date" display
- Return JSON array of stops: `{ stopIndex, customerId, poolId, customerName, address, poolName, poolType, sanitizerType, volumeGallons, gateCode, accessNotes, customerNotes, lastServiceDate, stopStatus }`
- If no route_day for today, return empty array `[]` (tech sees empty state)
- For non-tech roles (owner, office), accept an optional `?techId=` query param to view any tech's route

**2. Create server action `getTodayStops()`** in `src/actions/routes.ts`:
- Server-side equivalent of the API for SSR on the /routes page
- Uses `withRls()` to query route_days + customer/pool data
- Returns the same stop array shape

**3. Create server action `reorderStops(routeDayId, newOrder)`** in same file:
- Updates `route_days.stop_order` with new ordering
- Uses `withRls()` — but note: the route_days UPDATE policy is owner+office only
- For tech reordering: since techs cannot update route_days via RLS, store the reorder in Dexie `routeCache` locally only (the visual order changes but doesn't persist to server — Phase 4 will handle persistent reordering)
- Document this limitation clearly in comments

**4. Create server action `skipStop(routeDayId, customerId, reason)`** in same file:
- Creates a service_visit with `status: "skipped"` and `skip_reason` filled
- Uses `enqueueWrite()` pattern so it works offline

**5. Activate `prefetchTodayRoutes()`** in `src/lib/offline/sync.ts`:
- Replace the TODO stub with actual implementation
- Fetch from `/api/routes/today`
- Write each stop to `offlineDb.routeCache.bulkPut()` with 24-hour TTL
- Called on app open when online (existing wiring in initSyncListener)
