**Use the `frontend-design` skill for this task — per MEMORY.md user preference.**

**1. Audit and fix 44px tap targets:**
Review every interactive element in all field components. Any button, checkbox, input, or tappable area must have `min-h-[44px] min-w-[44px]`. Fix:
- Chemistry grid cells: ensure each input cell is at least 44px tall with generous padding
- Checklist checkboxes: ensure the touch target area (not just the checkbox icon) is 44px
- Tab triggers: ensure each tab is at least 44px tall
- Navigation button on stop cards: 44px
- Camera button: 44px (should already be, but verify)
- Complete/Skip buttons: 44px
- Drag handle on stop cards: 44px grip area
- Back arrow on stop workflow: 44px

**2. High-contrast OKLCH palette for outdoor visibility:**
- Review all text colors in field components against the dark background
- Ensure primary text has sufficient contrast ratio (minimum 7:1 for field use in sunlight)
- Status badges: use saturated colors (not muted pastels) for visibility
  - Complete: bright green `oklch(0.75 0.18 142)`
  - Skipped: bright amber `oklch(0.75 0.15 85)`
  - In Progress: bright blue `oklch(0.70 0.17 250)`
  - Upcoming: neutral muted
- Chemistry range colors: ensure red/yellow/green cells are distinguishable in sunlight
- LOW/HIGH badges: bold text, high contrast background

**3. Verify 3-tap rule:**
Map the most common actions and count taps from route list:
- Enter a chemistry reading: route list -> tap stop card (1) -> tap chemistry cell (2) -> enter value (3) = 3 taps OK
- Complete a checklist task: route list -> tap stop card (1) -> tap Tasks tab (2) -> tap checkbox (3) = 3 taps OK
- Snap a photo: route list -> tap stop card (1) -> tap Photos tab (2) -> tap camera (3) = 3 taps OK
- Mark all complete: route list -> tap stop card (1) -> tap Tasks tab (2) -> tap Mark All (3) = 3 taps OK
- Complete a stop: varies (must enter data first)
If any common action exceeds 3 taps, restructure the navigation to reduce tap count.

**4. Create tech settings page** (`src/app/(app)/settings/page.tsx`):
- Extend the existing settings page (or create if it's a placeholder) with:
  - **Maps App Preference:**
    - Radio group: Apple Maps | Google Maps
    - Stores preference in `localStorage` key `poolco-maps-pref`
    - Default: "apple" on iOS (detect via userAgent), "google" otherwise
  - **Display Name:** read-only view of profile name
  - **Role:** read-only view of user role

**5. Add `cursor-pointer` class** to all interactive elements (per Pending Todos from STATE.md). While this matters less on mobile, it improves desktop/tablet experience.

**6. Transitions and animations (Claude's discretion):**
- Tab switching: subtle fade transition (150ms)
- Stop card completion: brief green flash on status badge change
- Progress bar fill: smooth width transition (300ms ease)
- Photo thumbnail appear: fade-in (200ms)
- No heavy animations — performance over polish

Dark-first design system throughout. All changes must preserve existing functionality.
