**src/app/(app)/customers/[id]/page.tsx** — Server component:

Fetch customer with all related data using a relational query inside `withRls`:
```typescript
const customer = await withRls(token, (tx) =>
  tx.query.customers.findFirst({
    where: (c, { eq }) => eq(c.id, params.id),
    with: {
      pools: {
        with: {
          equipment: true,
          serviceVisits: {
            orderBy: (sv, { desc }) => [desc(sv.visited_at)],
          },
        },
      },
      assignedTech: {
        columns: { id: true, full_name: true },
      },
    },
  })
)
```

If customer is null, call `notFound()` from next/navigation.

Render the profile layout:
1. `<CustomerHeader>` — always visible above tabs
2. `<Tabs defaultValue="overview">` with 4 tabs: Overview, Pools, Equipment, History
3. TabsContent for each:
   - Overview: `<CustomerInlineEdit customer={customer} />`
   - Pools: `<PoolList pools={customer.pools} customerId={customer.id} />`
   - Equipment: `<EquipmentList pools={customer.pools} />` (equipment grouped by pool)
   - History: `<ServiceHistoryTimeline visits={allVisits} />` (flatten visits from all pools + direct customer visits). For now, render a placeholder: "No service history yet. Service records will appear here once field techs complete stops." (Phase 3 populates this.)

Also fetch the tech list for the inline edit's Assigned Tech select.
