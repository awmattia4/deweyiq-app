Build a vertical timeline per locked decisions:

**Props interface:**
```typescript
type ServiceVisit = {
  id: string
  visit_type: string | null  // "routine" | "repair" | "one_off"
  visited_at: string         // ISO date
  notes: string | null
  pool?: { id: string; name: string } | null
  tech?: { id: string; full_name: string } | null
  // Phase 3 will add: chemistry_readings, checklist_items, photos
}

interface ServiceHistoryTimelineProps {
  visits: ServiceVisit[]
}
```

**Filter chips** (per locked decision):
- Row of toggle chips at the top: "All", "Routine", "Repair", "One-off"
- `useState` for active filter
- Filter the visits list based on selected chip
- Chips styled with shadcn Badge or custom toggle buttons

**Timeline layout** (per locked decision):
- Vertical timeline with a line running down the left side
- Each visit is a card positioned to the right of the line
- Date marker (formatted: "Mar 5, 2026") on the left side of the line or as a section header when date changes
- Each card shows:
  - Visit type badge (Routine=blue, Repair=orange, One-off=purple)
  - Pool name (if associated with a specific pool)
  - Tech name who performed the service
  - Notes text
  - **Chemistry readings area** — inline by default (per locked decision). For Phase 2, show placeholder text: "Chemistry readings recorded during service" or render actual readings if the visit data includes them. Design the layout for: pH, Cl, Alk as primary inline values. Structure it so Phase 3 just passes data.
  - **Photo thumbnail strip** — horizontal strip below the card content. For Phase 2, render nothing if no photos. Structure the area so Phase 3 can add `visit.photos` array and thumbnails render in a horizontal scroll.

**Empty state:**
When visits array is empty, show:
- Calendar or clipboard icon (from lucide-react)
- "No service history yet"
- "Service records will appear here automatically as technicians complete stops."
- Subtle note: "Records include chemistry readings, service checklists, photos, and notes."

**Design notes:**
- Use the dark-first design system (existing Tailwind classes)
- Timeline line: `border-l-2 border-border` or similar
- Cards: subtle background (card), rounded, with left connector dot on the timeline line
- Responsive: on mobile, timeline is full-width (no side-by-side)
