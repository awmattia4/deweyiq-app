    Implement `getRevenueDashboard(startDate: string, endDate: string)` returning:
    ```typescript
    interface RevenueDashboardData {
      totalRevenue: number
      previousPeriodRevenue: number  // For trend calculation
      invoiceCount: number
      avgInvoiceValue: number
      outstandingAR: number  // Sum of unpaid sent invoices (sent but not paid)
      revenueByMonth: Array<{ month: string; revenue: number }>
      revenueByCustomer: Array<{
        customerId: string
        customerName: string
        totalRevenue: number
        invoiceCount: number
        lastPaymentDate: string | null
      }>
      revenueByTech: Array<{
        techId: string
        techName: string
        totalRevenue: number
        customerCount: number
      }>
    }
    ```

    **Query patterns (ALL must use LEFT JOIN + GROUP BY, no correlated subqueries per MEMORY.md):**

    a. **Total revenue + count + average**: Query paid invoices in date range, SUM total, COUNT, and calculate average. Also query the previous equivalent period for trend comparison (if current period is 30 days, previous is the 30 days before that).

    b. **Outstanding AR**: Query invoices WHERE status = 'sent' (sent but unpaid), SUM total. This is NOT filtered by date range — it's a current snapshot of what's owed.

    c. **Revenue by month**: Group paid invoices by TO_CHAR(paid_at, 'YYYY-MM'), order ascending. This feeds the AreaChart.

    d. **Revenue by customer**: LEFT JOIN invoices to customers, GROUP BY customer_id, ORDER BY SUM DESC. Include last paid_at date per customer using MAX(paid_at).

    e. **Revenue by tech**: LEFT JOIN invoices to customers (on customer_id), then LEFT JOIN customers to profiles (on assigned_tech_id). GROUP BY tech_id, ORDER BY SUM DESC. Count distinct customer_ids per tech. Label clearly: "Based on current tech assignment" (a customer reassigned after payment may shift revenue attribution).

    Role guard: only owner and office can access revenue data. Return empty data for tech/customer roles.

    Also implement `getCustomerRevenueDetail(customerId: string, startDate: string, endDate: string)` for the drill-down drawer:
    ```typescript
    interface CustomerRevenueDetail {
      customerName: string
      invoices: Array<{
        id: string
        invoiceNumber: string | null
        total: number
        paidAt: string | null
        billingModel: string | null
        status: string
      }>
      totalRevenue: number
      billingModelBreakdown: Array<{ model: string; total: number }>
    }
    ```
    Query paid invoices for the specific customer in the date range, grouped by billing_model for the breakdown.

    Also implement `exportRevenueCsv(startDate: string, endDate: string)` — owner-only CSV export of revenue by customer data. Follow the existing `exportFinancialCsv` pattern with `csvEscape` helper. Columns: Customer, Total Revenue, Invoice Count, Last Payment, Assigned Tech.
