    **1. `getTeamMetrics(startDate: string, endDate: string)`** — returns:
    ```typescript
    interface TechScorecardRow {
      techId: string
      techName: string
      // Speed/volume metrics
      completedStops: number
      daysWorked: number
      stopsPerDay: number  // completedStops / daysWorked
      avgStopMinutes: number  // average (completed_at - started_at) in minutes
      onTimeRate: number  // % of stops completed on scheduled_date
      // Quality metrics
      chemistryAccuracy: number  // % of readings within target range
      checklistCompletionRate: number  // % of checklist items completed across all visits
      photoRate: number  // % of visits with at least 1 photo
      // Trends (compared to previous equivalent period)
      stopsPerDayTrend: number  // positive = improvement
      avgStopMinutesTrend: number
      onTimeRateTrend: number
      chemistryAccuracyTrend: number
    }

    interface TeamMetricsData {
      techs: TechScorecardRow[]
      previousPeriodAvg: {
        stopsPerDay: number
        avgStopMinutes: number
        onTimeRate: number
        chemistryAccuracy: number
      }
    }
    ```

    **Query approach:**

    a. **Stops per day + days worked**: Query route_stops, COUNT completed stops, COUNT DISTINCT scheduled_date WHERE status='complete', GROUP BY tech_id. LEFT JOIN profiles.

    b. **Avg stop time**: Query route_stops WHERE status='complete' AND started_at IS NOT NULL. Calculate AVG(EXTRACT(EPOCH FROM (updated_at - started_at)) / 60) — using `updated_at` as proxy for completion time (completed_at is on service_visits, not route_stops). If `started_at` is NULL (stops before Phase 9), exclude from average. Note in the implementation: "Avg stop time only includes stops with started_at data (Phase 9+)."

    c. **Chemistry accuracy**: Two-step query (to avoid correlated subquery). Step 1: Fetch all service_visits in date range with tech_id and chemistry_readings. Step 2: In application code, import `classifyReading` from `@/lib/chemistry/targets`, run it on each reading parameter, count how many are "ok"/"balanced" vs total readings. This gives `chemistryAccuracy = inRangeCount / totalReadingsCount * 100` per tech.

    d. **Checklist completion rate**: From service_visits.checklist_completion JSONB — count completed:true items / total items across all visits per tech.

    e. **Photo rate**: From service_visits.photo_urls — count visits where photo_urls array has length > 0 / total visits per tech.

    f. **Trends**: Run the same queries for the previous equivalent period. Calculate delta per metric.

    Role guard: owner and office can see all techs. Tech can only see their own data (use `getTechScorecard` below).

    **2. `getTechScorecard(techId: string, startDate: string, endDate: string)`** — returns a single `TechScorecardRow` for the specified tech. Used by the tech self-view. Role guard: tech can only query their own ID (verify token.sub === techId), owner/office can query any tech.

    **3. `getPayrollPrep(startDate: string, endDate: string)`** — returns:
    ```typescript
    interface PayrollRow {
      techId: string
      name: string
      email: string
      payType: string  // 'per_stop' | 'hourly'
      completedStops: number
      hoursWorked: number | null  // estimated from stop times if available
      payRate: number
      basePay: number  // stops * rate (per_stop) or hours * rate (hourly)
      upsellCommissions: number  // commission on completed WOs flagged by this tech
      totalGross: number  // basePay + upsellCommissions
    }
    ```

    **Payroll query approach:**
    a. Fetch profiles with pay_type and pay_rate for all techs/owner in the org.
    b. Count completed stops per tech in date range.
    c. For hourly techs: estimate hours from SUM of stop durations (started_at to completed_at/updated_at). If no started_at data, show null.
    d. Calculate basePay: per_stop techs get `completedStops * payRate`, hourly techs get `hoursWorked * payRate`.
    e. Upsell commissions: Query work_orders WHERE flagged_by_tech_id = tech AND status IN ('complete', 'invoiced') AND completed_at in date range. SUM the invoice totals for those WOs, multiply by org's `wo_upsell_commission_pct`. Use adminDb for this cross-table query to avoid RLS complexity.
    f. totalGross = basePay + upsellCommissions.

    Handle null pay_rate gracefully: if pay_rate is null, basePay = 0 and add a `payRateConfigured: boolean` flag so UI can show "Configure pay rate" prompt.

    Role guard: owner only.

    **4. `exportPayrollCsv(startDate: string, endDate: string)`** — owner-only. Columns: Employee Name, Employee Email, Pay Type, Period Start, Period End, Completed Stops, Hours Worked, Pay Rate, Base Pay, Upsell Commissions, Total Gross Pay. Follow existing csvEscape pattern.

    **5. `exportTeamCsv(startDate: string, endDate: string)`** — owner-only CSV of scorecard data. Columns: Tech Name, Stops/Day, Avg Stop Time (min), On-Time Rate %, Chemistry Accuracy %, Checklist Rate %, Photo Rate %.

    **6. Add to `src/actions/company-settings.ts`** (or create a new action if appropriate):
    `updateTechPayConfig(techId: string, payType: string, payRate: number)` — updates profiles.pay_type and profiles.pay_rate. Owner-only. Uses withRls with the owner's token (profiles_update_policy allows owner to update any profile in their org).
