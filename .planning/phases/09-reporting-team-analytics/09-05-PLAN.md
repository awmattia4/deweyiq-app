    **2. Append to `src/actions/reporting.ts`:**

    Implement `getProfitabilityAnalysis(startDate: string, endDate: string)` returning:
    ```typescript
    interface PoolProfitability {
      poolId: string
      poolName: string
      customerId: string
      customerName: string
      // Revenue side
      recurringRevenue: number  // paid invoices for this customer in period
      // Cost side
      totalChemicalCost: number  // sum of (dosing_amount * cost_per_unit) across visits
      visitCount: number
      avgCostPerVisit: number
      // Margin
      margin: number  // revenue - chemicalCost
      marginPct: number  // margin / revenue * 100 (handle division by zero)
      // Flagging
      isFlagged: boolean  // marginPct < threshold
      flagSeverity: "red" | "yellow" | null  // red = negative margin, yellow = below threshold
      // Data quality
      hasEstimatedCosts: boolean  // true if any visits used re-derived dosing (no dosing_amounts stored)
    }

    interface TechDosingCost {
      techId: string
      techName: string
      totalChemicalCost: number
      visitCount: number
      avgCostPerVisit: number
      // Per-chemical breakdown
      costByChemical: Array<{ chemical: string; totalCost: number; totalAmount: number; unit: string }>
    }

    interface ProfitabilityData {
      pools: PoolProfitability[]
      flaggedPools: PoolProfitability[]  // filtered subset where isFlagged = true
      techCosts: TechDosingCost[]
      thresholdPct: number  // the configured margin threshold
      totalChemicalCost: number
      totalRecurringRevenue: number
      overallMarginPct: number
    }
    ```

    **Query approach (complex — multiple steps, all LEFT JOIN):**

    a. **Fetch org settings**: Get `chem_profit_margin_threshold_pct` for the threshold. Use adminDb.

    b. **Fetch chemical products with cost_per_unit**: Query all active chemical_products for the org. Build a Map<productId, costPerUnit>.

    c. **Fetch service visits with dosing data**: Query service_visits in date range with pool_id, tech_id, dosing_amounts JSONB, chemistry_readings JSONB. LEFT JOIN pools for pool name, LEFT JOIN customers for customer name.

    d. **Calculate chemical cost per visit**:
       - If `dosing_amounts` is populated (Phase 9+ visits): sum(dose.amount * productCostMap.get(dose.productId))
       - If `dosing_amounts` is null (historical visits): re-derive using `generateDosingRecommendations()` from `@/lib/chemistry/dosing` with the visit's chemistry_readings and pool volume. Mark as estimated. This is CPU-intensive — limit to last 500 visits per pool if performance is a concern.
       - If cost_per_unit is null for a product, skip that chemical's cost (show $0 with a note).

    e. **Aggregate per pool**: GROUP visits by pool_id, SUM chemical costs, COUNT visits.

    f. **Fetch revenue per customer**: Query paid invoices in date range, grouped by customer_id. Map to pools (a customer may have multiple pools — distribute revenue evenly or attribute to each pool proportionally based on visit count).

    g. **Calculate margin**: margin = revenue - chemicalCost. marginPct = (margin / revenue) * 100. Handle revenue = 0 (marginPct = -100 if costs > 0, 0 if costs also 0).

    h. **Flag pools**: isFlagged = marginPct < thresholdPct. flagSeverity = "red" if margin < 0, "yellow" if 0 <= marginPct < threshold.

    i. **Aggregate per tech**: GROUP visits by tech_id, SUM chemical costs, break down by chemical type.

    Role guard: owner only.

    Also implement `exportProfitabilityCsv(startDate: string, endDate: string)` — columns: Pool, Customer, Revenue, Chemical Cost, Visits, Avg Cost/Visit, Margin, Margin %, Flagged, Estimated. Owner-only.
