  <action>
    Append to `src/actions/reporting.ts` (file created by Plan 02 and already contains revenue actions).

    Implement `getOperationsMetrics(startDate: string, endDate: string)` returning:
    ```typescript
    interface OperationsMetricsData {
      // Company-wide summary
      totalStops: number
      completedStops: number
      skippedStops: number
      missedStops: number  // scheduled but not completed and date has passed
      completionRate: number  // completedStops / totalStops * 100
      previousCompletionRate: number  // for trend
      onTimeRate: number  // % of completed stops where completed_at::date = scheduled_date
      previousOnTimeRate: number  // for trend
      // Per-tech breakdown
      techMetrics: Array<{
        techId: string
        techName: string
        totalStops: number
        completedStops: number
        skippedStops: number
        missedStops: number
        completionRate: number
        onTimeRate: number
      }>
      // Daily completion data for chart (stacked bar)
      dailyCompletion: Array<{
        date: string  // YYYY-MM-DD
        completed: number
        skipped: number
        missed: number
      }>
    }
    ```

    **Query pattern (LEFT JOIN + GROUP BY, no correlated subqueries):**

    a. **Per-tech metrics**: Query route_stops LEFT JOIN profiles (on tech_id), filtered by date range and scheduled_date < today (only count past stops). Use COUNT with FILTER:
       - `completed`: status = 'complete'
       - `skipped`: status = 'skipped'
       - `missed`: status NOT IN ('complete', 'skipped', 'holiday') AND scheduled_date < today
       - Exclude `holiday` status from totals
       GROUP BY tech_id, profiles.full_name

    b. **On-time rate**: "On-time" = stop completed on its scheduled_date (completed_at::date = scheduled_date OR updated_at::date = scheduled_date for stops without explicit completed_at). Calculate as an additional metric at both company-wide and per-tech levels. Include `onTimeRate` and `previousOnTimeRate` fields in the returned data.

    c. **Daily completion for chart**: Query route_stops grouped by scheduled_date, same FILTER counts. Only include dates where scheduled_date < today. ORDER BY scheduled_date ASC. This feeds the stacked bar chart.

    d. **Previous period** for trend: Calculate the equivalent period before startDate and run same completion rate and on-time rate queries.

    Role guard: owner and office only.

    Also implement `exportOperationsCsv(startDate: string, endDate: string)` — owner-only CSV with columns: Tech Name, Total Stops, Completed, Skipped, Missed, Completion Rate %, On-Time Rate %.
