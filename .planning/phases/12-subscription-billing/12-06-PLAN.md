  <action>
    Create `src/lib/billing-status.ts`:

    Export type `RestrictionReason = "past_due_grace_expired" | "unpaid" | "trial_ended" | null`

    Export function `getAccountRestrictionReason(subscription)`:
    - Takes the subscription row (or null)
    - Returns RestrictionReason:
      - If no subscription: return "trial_ended" (org exists but never set up billing)
      - If status "unpaid": return "unpaid"
      - If status "past_due" AND grace_period_ends_at AND grace_period_ends_at < now: return "past_due_grace_expired"
      - If status "canceled" AND current_period_end < now: return "trial_ended" (treating expired canceled as ended)
      - If status "trialing" AND trial_ends_at AND trial_ends_at < now: return "trial_ended"
      - Otherwise: return null (no restriction)

    Export function `isAccountRestricted(subscription)`:
    - Returns boolean: `getAccountRestrictionReason(subscription) !== null`

    NOTE: This is a shared utility, not a server action. It runs on the server during SSR and in server actions.
  </action>
  <verify>
    1. Module compiles
    2. All restriction scenarios covered
    3. Returns null for active/trialing accounts within limits
  </verify>
