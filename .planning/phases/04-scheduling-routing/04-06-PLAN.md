  <action>
    **Create `src/actions/optimize.ts`:**

    Server action that calls OpenRouteService optimization API. CRITICAL: Call from server action, NOT client -- ORS_API_KEY must not be exposed.

    Following RESEARCH.md Pattern 5 with locked stop handling:

    **`optimizeRoute(techId: string, date: string)` server action:**

    1. Auth check: getCurrentUser, owner/office only.
    2. Fetch all route_stops for this tech+date from DB (via withRls).
    3. Separate locked and unlocked stops:
       - `lockedStops = stops.filter(s => s.position_locked)`
       - `unlockedStops = stops.filter(s => !s.position_locked)`
    4. Filter out stops without lat/lng (cannot optimize without coordinates). Log warning for any missing.
    5. Build ORS optimization request:
       ```ts
       const jobs = unlockedStops.map((stop, idx) => ({
         id: idx,
         location: [stop.lng, stop.lat] as [number, number],
         // Include time windows if set
         ...(stop.window_start && stop.window_end ? {
           time_windows: [[timeToSeconds(stop.window_start), timeToSeconds(stop.window_end)]]
         } : {})
       }))

       // Tech starting location: use the tech's current position if available,
       // otherwise use the first stop's location as start
       const startLocation = unlockedStops[0]
         ? [unlockedStops[0].lng, unlockedStops[0].lat]
         : [lockedStops[0]?.lng ?? -96, lockedStops[0]?.lat ?? 39]

       const response = await fetch("https://api.openrouteservice.org/optimization", {
         method: "POST",
         headers: {
           Authorization: process.env.ORS_API_KEY!,
           "Content-Type": "application/json",
         },
         body: JSON.stringify({
           jobs,
           vehicles: [{
             id: 0,
             start: startLocation,
             profile: "driving-car",
           }],
         }),
       })
       ```
    6. Parse ORS response:
       - `result.routes[0].steps` contains optimized job sequence
       - `result.summary.duration` gives total optimized drive time in seconds
    7. Re-insert locked stops at their original positions:
       - Build a new full stop order array
       - Place locked stops at their original sort_index positions
       - Fill gaps with unlocked stops in the ORS-optimized order
    8. Calculate current total drive time (approximate from current order using Haversine distances between consecutive stops at average speed 30mph) for comparison.
    9. Return:
       ```ts
       interface OptimizationResult {
         success: boolean
         error?: string
         currentOrder: Array<{ id: string; customerName: string; address: string | null; sortIndex: number; locked: boolean }>
         optimizedOrder: Array<{ id: string; customerName: string; address: string | null; sortIndex: number; locked: boolean }>
         currentDriveTimeMinutes: number
         optimizedDriveTimeMinutes: number
         timeSavedMinutes: number
         stopsWithoutCoordinates: number  // warn user if any
       }
       ```

    **Helper functions in the same file:**

    - `timeToSeconds(timeStr: string): number` -- convert "HH:MM:SS" time to seconds from midnight
    - `haversineDistance(lat1, lng1, lat2, lng2): number` -- distance in km between two points
    - `estimateDriveTimeMinutes(stops: Array<{lat, lng}>): number` -- sum of haversine distances / average speed

    **Error handling:**
    - If ORS returns non-200, return `{ success: false, error: "Optimization service unavailable" }`
    - If no stops have coordinates, return `{ success: false, error: "No stops have geocoded coordinates" }`
    - If ORS free tier limit hit, return `{ success: false, error: "Optimization quota exceeded, try again later" }`

    **`applyOptimizedOrder(techId: string, date: string, optimizedStopIds: string[])` server action:**

    After user clicks "Apply":
    - Update sort_index on each route_stop to match the new order
    - Revalidate /schedule path
    - Return success

    This is separate from optimizeRoute so the user can preview before applying (per user decision).
