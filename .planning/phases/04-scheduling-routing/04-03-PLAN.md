    **1. Create `src/components/map/map-client.tsx`:**

    A shared MapLibre GL JS wrapper component using `next/dynamic` SSR-safe pattern.

    CRITICAL: MapLibre accesses `window` on import. This component MUST be loaded via `next/dynamic` with `ssr: false` by consuming components. The component itself is a "use client" component.

    ```tsx
    "use client"
    import { useEffect, useRef, useCallback, forwardRef, useImperativeHandle } from "react"
    import maplibregl from "maplibre-gl"
    import "maplibre-gl/dist/maplibre-gl.css"
    ```

    Props:
    - `center?: [number, number]` -- [lng, lat] initial center (default to a US center like [-96, 39])
    - `zoom?: number` -- initial zoom (default 11)
    - `className?: string` -- container class
    - `onMapReady?: (map: maplibregl.Map) => void` -- callback when map is initialized

    Use ref pattern:
    - `mapContainer` ref for the div element
    - `map` ref for the MapLibre instance
    - Initialize map in useEffect with cleanup (map.remove() + null ref)
    - Tile source: `https://api.maptiler.com/maps/streets-v2-dark/style.json?key=${process.env.NEXT_PUBLIC_MAPTILER_KEY}` (dark variant matches dark-first design system)
    - If NEXT_PUBLIC_MAPTILER_KEY is not set, use a fallback open tile source or show a placeholder message

    Export a `MapClientHandle` interface that exposes the map instance for imperative control.

    **2. Create `src/components/schedule/route-map.tsx`:**

    "use client" component that renders the map with plotted stops and route line.

    Import MapClient via `next/dynamic`:
    ```tsx
    import dynamic from "next/dynamic"
    const MapClient = dynamic(() => import("@/components/map/map-client").then(m => m.MapClient), { ssr: false })
    ```

    Props:
    - `stops: ScheduleStop[]` -- ordered stops with lat/lng
    - `selectedStopId?: string` -- highlight the selected stop

    Behavior:
    - On map ready, add markers for each stop (numbered pins matching sort_index)
    - Draw a GeoJSON LineString route line through stops in sort_index order (only stops with lat/lng)
    - Completed stops: grayed out markers
    - Locked stops: visually distinct (lock icon or different marker color)
    - When stops are reordered (stops prop changes), update the route line and marker numbers
    - Fit bounds to show all stops when stops change
    - Click a stop marker to highlight it (selectedStopId callback)
    - Use OKLCH colors from the project's dark-first palette for marker colors

    Define `ScheduleStop` interface (or import from schedule.ts):
    ```ts

      address: string | null
      poolName: string
      sortIndex: number
      positionLocked: boolean
      status: string
      lat: number | null
      lng: number | null
    }
    ```

    - `onSelectStop?: (stopId: string) => void` -- select for map highlight
