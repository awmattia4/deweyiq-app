### NOTIF-04 Scope Note

The raw requirement text says "whether they route to email, in-app notification, or SMS per alert category." The Context document scoped Phase 5 to enabled/disabled toggles per notification type (not per-channel routing). Per-channel routing for office alerts (email vs. SMS vs. in-app per alert type) was never planned for Phase 5. The implemented settings cover: which customer notification channels are on/off (pre-arrival SMS, pre-arrival email, service report email) and which alert types are enabled/disabled (missed stop, declining chemistry, incomplete data). This is a narrower scope than the full NOTIF-04 requirement text, but it matches the Phase 5 context decisions.

**Gap status:** NOTIF-04 is scoped-partial — what was built matches the Phase 5 context decisions. Full per-category channel routing is a future enhancement outside Phase 5 scope.
