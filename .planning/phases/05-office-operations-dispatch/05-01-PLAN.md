<task type="auto">
  <name>Task 2: Create send-pre-arrival Supabase Edge Function with Twilio SMS and Resend email fallback</name>
  <files>
    supabase/functions/send-pre-arrival/index.ts
  </files>
  <action>
    Create `supabase/functions/send-pre-arrival/index.ts` following the exact same structural pattern as `supabase/functions/send-service-report/index.ts`:
    - Deno.serve handler with CORS OPTIONS handling
    - Request body interface:
      ```typescript
      interface RequestBody {
        orgId: string
        techName: string
        stops: Array<{
          stopId: string           // route_stop id for pre_arrival_sent_at update
          customerName: string
          customerPhone: string | null
          customerEmail: string | null
          stopNumber: number       // position in route (e.g., 3)
          notificationsEnabled: boolean
        }>
      }
      ```
    - Read env vars: TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_PHONE_NUMBER, RESEND_API_KEY
    - For each stop where notificationsEnabled is true:
      - If customerPhone exists: Send SMS via Twilio REST API using `fetch` to `https://api.twilio.com/2010-04-01/Accounts/${accountSid}/Messages.json` with HTTP Basic auth (`btoa(${accountSid}:${authToken})`) and URLSearchParams body. Message: `Hi ${customerName}, your pool tech ${techName} is heading your way. You're stop #${stopNumber} on today's route.`
      - Else if customerEmail exists: Send email fallback via Resend REST API (same pattern as send-service-report) with a simple HTML message matching the SMS content.
      - After successful send, update route_stops.pre_arrival_sent_at via Supabase admin client (same pattern as send-service-report updating email_sent_at).
    - Return JSON response with sent count and any errors.
    - CRITICAL: Do NOT import the Twilio npm package — it does not work in Deno. Use raw fetch + URLSearchParams + btoa for HTTP Basic auth. This is the verified community pattern.
    - CRITICAL: Use `jsr:@supabase/supabase-js@2` import (not esm.sh) — matches existing project pattern.

    The Edge Function does NOT need to filter by notificationsEnabled — the calling server action will pre-filter and only pass eligible stops. But include the check as a safety net.
  </action>

    3. File uses raw `fetch` for Twilio (no twilio npm import)
    4. File uses `btoa` for HTTP Basic auth
    5. File handles CORS OPTIONS
    6. `npm run build` still succeeds (Edge Function is excluded from Node.js tsconfig per Phase 3 decision)
  </verify>
