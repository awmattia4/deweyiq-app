<task type="auto">
  <name>Task 2: Wire warn-but-allow enforcement into stop completion flow</name>
  <files>
    src/actions/visits.ts

  <action>
    Update the completeStop flow in `src/actions/visits.ts` to check service requirements and enforce the warn-but-allow pattern:

    1. Before completing the stop, fetch org_settings via getOrgSettings()
    2. If org_settings has required_chemistry_by_sanitizer:
       - Look up the pool's sanitizer_type
       - Get the required chemistry params for that sanitizer type
       - Check if the submitted chemistry_readings has non-null values for all required params
       - Build a list of missing required params
    3. If org_settings has required_checklist_task_ids:
       - Check if the submitted checklist_completion marks all required tasks as completed
       - Build a list of missing required tasks
    4. Accept a new parameter in the completeStop input: `overrideWarnings: boolean` (default false)
    5. If there are missing requirements AND overrideWarnings is false:
       - Return `{ success: false, warnings: { missingChemistry: [...], missingChecklist: [...] } }`
       - The client shows a warning dialog listing the missing items with two buttons: "Go Back" and "Complete Anyway"
       - "Complete Anyway" re-calls completeStop with overrideWarnings: true
    6. If overrideWarnings is true (tech chose to override):
       - Complete the stop normally
       - Generate an "incomplete_data" alert:
         - Insert into alerts table with alert_type: "incomplete_data"
         - Title: "{customerName}'s service has incomplete data (tech override)"
         - Severity: "warning"
         - metadata: { customerId, poolId, techId, missingChemistry: [...], missingChecklist: [...] }
    7. If there are NO missing requirements: complete normally, no alert generated

    CRITICAL: This is "warn but allow" — NEVER throw an error or prevent completion. The override path must always succeed. Field conditions (broken test kit, equipment access issues) make hard blocks frustrating for techs.

    The completion modal UI (likely in src/components/schedule/completion-modal.tsx or similar) needs to handle the warnings response:
    - On { success: false, warnings } response: show a warning dialog listing missing items
    - Warning dialog has: list of missing chemistry/checklist items, "Go Back" button, "Complete Anyway" button
    - "Complete Anyway" calls completeStop again with overrideWarnings: true

    Update the completion modal/dialog component to handle this new flow. Check which component handles the completion confirmation (could be in src/components/schedule/ or src/components/routes/).
  </action>

    6. Completion modal shows warning dialog for missing items
